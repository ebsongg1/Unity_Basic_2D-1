# 켜고 끄는 상태를 가진 버튼, Toggle

## Toggle이란 무엇인가?

방의 전등 스위치를 생각해봅시다. 스위치를 한 번 누르면 불이 '켜지고(On)', 다시 누르면 '꺼집니다(Off)'. 이처럼 **Toggle(토글)** 은 단발적인 동작을 실행하는 일반 버튼과 달리, '켜짐'과 '꺼짐'이라는 두 가지 상태를 가지고 그 상태를 유지하는 특별한 종류의 버튼입니다.

정의하자면, `Toggle`은 사용자의 클릭에 따라 **On/Off 상태가 전환되고 유지**되는 상호작용 컴포넌트입니다. 주로 게임 설정의 '전체 화면 모드 On/Off', '음소거 On/Off'처럼 켜고 끄는 옵션을 선택하는 데 사용됩니다. 또한 여러 개의 토글을 그룹으로 묶어 '라디오 버튼'처럼 여러 선택지 중 하나만 고르게 하는 기능도 구현할 수 있습니다.

`Toggle`은 보통 배경이 되는 `Background` 이미지와, On 상태일 때 나타나는 체크마크 `Checkmark` 이미지, 그리고 토글의 설명을 담는 `Label` 텍스트로 구성됩니다.

## 핵심 요약

-   Toggle은 **On/Off 두 가지 상태를 가지는** 특수한 버튼입니다.
-   **Is On** 속성을 통해 현재 토글의 상태를 확인하거나 변경할 수 있습니다.
-   **On Value Changed()** 이벤트를 통해 토글의 상태가 변경되었을 때 특정 함수를 호출할 수 있습니다.
-   **Toggle Group** 컴포넌트를 사용하면 여러 개의 토글을 묶어 하나만 선택 가능한 **라디오 버튼**으로 만들 수 있습니다.

## 세부 개념

### 1. Toggle 컴포넌트의 구조와 주요 속성

`Toggle`을 생성하면 `Background`, `Checkmark`, `Label` 자식 오브젝트들이 함께 만들어집니다.

-   **Toggle (Root)**: `Toggle` 컴포넌트가 있는 루트 오브젝트입니다.
    -   **Background**: 토글의 배경이 되는 이미지입니다. Off 상태일 때 주로 보입니다.
    -   **Checkmark**: On 상태임을 나타내는 체크 표시 이미지입니다. `Is On`이 `true`일 때 활성화됩니다.
    -   **Label**: 이 토글이 어떤 옵션인지 설명하는 텍스트입니다.

**Public Field (Toggle 컴포넌트)**

| 항목 | 설명 |
| :--- | :--- |
| `Interactable` | 토글을 사용자가 조작할 수 있는지 여부를 결정합니다. |
| `Transition` | `Button`과 동일하게, 입력에 따른 시각적 변화를 설정합니다. |
| `Graphic` | `Checkmark` 이미지를 지정합니다. `Is On` 상태에 따라 이 그래픽의 활성화 여부가 결정됩니다. |
| `Group` | 이 토글이 속할 `Toggle Group`을 지정합니다. 라디오 버튼 기능을 만들 때 사용합니다. |
| `Is On` | 토글의 현재 상태를 나타내는 `bool` 값입니다. `true`이면 On, `false`이면 Off 상태입니다. 스크립트에서 이 값을 읽거나 설정하여 토글을 제어합니다. |

### 2. On Value Changed (Boolean) 이벤트

`Toggle`의 상태가 변경될 때(On -> Off 또는 Off -> On) 호출되는 이벤트입니다.

-   `bool` 타입의 매개변수를 받는 함수를 연결할 수 있으며, 이 매개변수에는 변경된 `Is On` 상태(`true` 또는 `false`)가 자동으로 전달됩니다.
-   이를 통해 토글 상태 변화에 따른 실제 동작(예: 소리 켜기/끄기)을 즉시 실행할 수 있습니다.

### 3. Toggle Group (토글 그룹)

여러 개의 토글을 하나의 그룹으로 묶어, 그중 **단 하나만 On 상태가 되도록** 강제하는 컴포넌트입니다. 흔히 '라디오 버튼'이라고 불리는 기능을 구현할 때 사용됩니다.

-   빈 `GameObject`에 `Toggle Group` 컴포넌트를 추가하여 그룹을 만듭니다.
-   각 `Toggle` 컴포넌트의 `Group` 속성에 이 `Toggle Group` 오브젝트를 할당합니다.
-   `Allow Switch Off` 옵션을 `true`로 하면, 켜져 있는 토글을 다시 눌러서 그룹 전체를 Off 상태로 만들 수 있습니다. `false`이면 그룹 내에 항상 하나는 켜져 있어야 합니다.

-   **활용 예시**: 그래픽 품질을 '상', '중', '하' 중에서 하나만 고르게 할 때. '상', '중', '하'에 해당하는 3개의 토글을 만들고 하나의 `Toggle Group`으로 묶어줍니다.

## 코드 예제

```csharp
// SettingsManager.cs
// Toggle을 이용해 음소거 기능과 전체 화면 설정을 제어하는 스크립트

using UnityEngine;
using UnityEngine.UI; // Toggle을 사용하기 위해 필요

public class SettingsManager : MonoBehaviour
{
    public Toggle muteToggle;
    public Toggle fullscreenToggle;

    void Start()
    {
        // 게임 시작 시, 현재 설정 상태를 각 토글의 Is On 상태와 동기화
        if (muteToggle != null)
        {
            muteToggle.isOn = AudioListener.pause; // 현재 음소거 상태
        }
        if (fullscreenToggle != null)
        {
            fullscreenToggle.isOn = Screen.fullScreen; // 현재 전체 화면 상태
        }

        // 각 토글의 OnValueChanged 이벤트에 리스너를 등록
        muteToggle.onValueChanged.AddListener(SetMuteState);
        fullscreenToggle.onValueChanged.AddListener(SetFullscreenState);
    }

    // 음소거 토글의 상태가 변하면 호출될 함수
    // 매개변수 isMuted에는 토글의 새로운 isOn 상태가 전달됨
    public void SetMuteState(bool isMuted)
    {
        AudioListener.pause = isMuted;
        Debug.Log("음소거 상태: " + isMuted);
    }

    // 전체 화면 토글의 상태가 변하면 호출될 함수
    public void SetFullscreenState(bool isFullscreen)
    {
        Screen.fullScreen = isFullscreen;
        Debug.Log("전체 화면 상태: " + isFullscreen);
    }
}
```

## 활용 예시

'배경음악 켜고 끄기' 토글을 만들어 봅시다.

1.  `Hierarchy`에서 `UI -> Toggle`을 생성합니다. `Label` 텍스트를 "배경음악"으로 수정합니다.
2.  배경음악을 재생하는 `AudioSource`가 있는 `GameObject`(예: `SoundManager`)를 준비합니다.
3.  `SoundManager`에 아래와 같은 `public` 함수를 작성합니다.
    ```csharp
    public void ToggleBGM(bool isOn)
    {
        if (isOn)
        {
            bgmAudioSource.Play();
        }
        else
        {
            bgmAudioSource.Pause();
        }
    }
    ```
4.  다시 `Toggle`을 선택하고, 인스펙터의 `On Value Changed (Boolean)` 리스트에 `+`를 누릅니다.
5.  `SoundManager` 오브젝트를 슬롯에 드래그 앤 드롭합니다.
6.  함수 선택 드롭다운 메뉴에서 `SoundManager -> ToggleBGM (bool)`을 선택합니다.
7.  게임을 실행하고 토글을 클릭하면, 토글의 `Is On` 상태가 `ToggleBGM` 함수의 `isOn` 매개변수로 전달되어 배경음악이 켜지거나 꺼집니다.

## Note: 주의사항

-   **이벤트 매개변수**: `On Value Changed` 이벤트는 `bool` 타입의 동적(Dynamic) 매개변수만 지원합니다. 인스펙터에서 정적(Static) 값을 전달하는 것은 의미가 없습니다.
-   **초기 상태 동기화**: 게임 시작 시, 토글의 시각적 상태(`Is On`)와 실제 시스템의 설정 상태(예: `Screen.fullScreen`)가 일치하도록 초기화 코드를 작성해주는 것이 중요합니다. 그렇지 않으면 UI와 실제 상태가 다르게 보이는 문제가 발생할 수 있습니다.
-   **라디오 버튼과 `Allow Switch Off`**: `Toggle Group`을 사용할 때, 사용자가 반드시 여러 옵션 중 하나를 선택해야 하는 경우(예: 난이도 선택) `Allow Switch Off`를 `false`로 설정해야 합니다.

## Further Study: 확장 학습

-   **Toggle Group의 확장**: `Toggle Group`은 기본적으로 어떤 토글이 선택되었는지 알려주는 이벤트가 없습니다. `Toggle Group`을 상속받아 현재 활성화된 토글을 알려주거나, 선택된 토글의 인덱스를 반환하는 등 확장된 기능을 가진 커스텀 그룹을 만들어 보세요.
-   **Tab Menu 구현**: 여러 개의 토글 버튼과 여러 개의 패널을 조합하여, 특정 토글을 선택했을 때 그에 해당하는 패널만 활성화되는 탭 메뉴 시스템을 구현해 보세요. `Toggle Group`과 `On Value Changed` 이벤트를 효과적으로 활용하는 좋은 연습이 됩니다.
