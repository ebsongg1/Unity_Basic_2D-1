# 자식들을 자동으로 정렬하는, Layout Group

## Layout Group이란 무엇인가?

책장에 책을 정리하는 상황을 상상해봅시다. 책을 아무렇게나 쌓아두면 뒤죽박죽이 되어 원하는 책을 찾기 어렵습니다. 하지만 '가나다순', '분야별' 같은 규칙에 따라 책장에 차곡차곡 정리하면 아주 깔끔해집니다. 이때 책장은 책들을 담고, 일정한 규칙에 따라 정렬해주는 '틀'의 역할을 합니다.

UI에서 **Layout Group(레이아웃 그룹)** 은 바로 이 '책장'과 같습니다. Layout Group 컴포넌트가 적용된 UI 요소의 자식으로 들어오는 다른 UI 요소(책)들은, 우리가 정해준 규칙(가로/세로/격자)에 따라 자동으로 위치가 정렬됩니다.

정의하자면, `Layout Group`은 자식 `RectTransform`들의 위치, 정렬, 그리고 선택적으로 크기까지 자동으로 제어하여 깔끔한 UI 레이아웃을 손쉽게 만들어주는 강력한 컴포넌트 시스템입니다. `Horizontal Layout Group`, `Vertical Layout Group`, `Grid Layout Group` 세 가지가 있으며, 동적으로 아이템이 추가되거나 삭제되는 목록, 인벤토리, 메뉴 등을 만들 때 수동으로 위치를 계산할 필요가 없게 해주는 필수 기능입니다.

## 핵심 요약

-   Layout Group은 **자식 UI 요소들을 자동으로 정렬**하는 컴포넌트입니다.
-   **종류**: `Horizontal`(수평 정렬), `Vertical`(수직 정렬), `Grid`(격자 정렬) 세 가지가 있습니다.
-   **Padding(여백)** 과 **Spacing(간격)** 속성을 통해 전체적인 틀과 요소 간의 거리를 쉽게 조절할 수 있습니다.
-   `Content Size Fitter`, `Layout Element` 컴포넌트와 함께 사용하면 내용에 따라 크기가 변하는 유동적인 UI를 만들 수 있습니다.

## 세부 개념

### 1. 공통 속성

모든 Layout Group은 다음과 같은 공통 속성을 가집니다.

| 항목 | 설명 |
| :--- | :--- |
| `Padding` | Layout Group의 경계(테두리)와 내부 자식 요소들 사이의 전체적인 안쪽 여백을 설정합니다. (Left, Right, Top, Bottom) |
| `Spacing` | 자식 요소들 사이의 간격을 설정합니다. |
| `Child Alignment` | 자식 요소들이 Layout Group 내에서 공간이 남을 경우, 어느 곳에 정렬될지를 결정합니다. (예: `UpperLeft`, `MiddleCenter`) |
| `Control Child Size` | `true`이면 Layout Group이 자식 요소들의 너비나 높이를 제어합니다. |
| `Use Child Scale` | 자식의 스케일 값을 레이아웃 계산에 포함할지 결정합니다. |
| `Child Force Expand` | `true`이면 자식 요소들이 Layout Group의 가용 공간을 채우기 위해 강제로 확장됩니다. |

### 2. Layout Group의 종류

#### 1) Horizontal Layout Group
자식 요소들을 왼쪽에서 오른쪽으로, 하나의 수평 행에 차례대로 정렬합니다.

-   **활용 예시**: 게임 상단의 버프 아이콘 목록, 상점의 가로 스크롤 아이템 목록 등.

#### 2) Vertical Layout Group
자식 요소들을 위에서 아래로, 하나의 수직 열에 차례대로 정렬합니다.

-   **활용 예시**: 설정 메뉴 항목, 채팅창의 대화 내용, 수직 스크롤 미션 목록 등.

#### 3) Grid Layout Group
자식 요소들을 바둑판식 격자(Grid) 형태로 정렬합니다. 가장 많은 옵션을 가지고 있습니다.

**Public Field (Grid Layout Group)**

| 항목 | 설명 |
| :--- | :--- |
| `Cell Size` | 격자의 각 칸(Cell)의 고정 크기를 지정합니다. |
| `Constraint` | 그리드의 형태를 제한하는 방식입니다. `Flexible`(유연), `Fixed Column Count`(열 개수 고정), `Fixed Row Count`(행 개수 고정) 중 선택합니다. |
| `Constraint Count` | `Constraint`가 열/행 고정일 때, 그 개수를 지정합니다. |
| `Start Corner` | 첫 번째 자식 요소가 시작될 모서리를 결정합니다. (예: `UpperLeft`) |
| `Start Axis` | 격자를 채워나가는 주축을 결정합니다. `Horizontal`이면 가로로 채우다 꽉 차면 다음 줄로, `Vertical`이면 세로로 채우다 꽉 차면 다음 열로 넘어갑니다. |

-   **활용 예시**: MMORPG의 6x5 인벤토리 창, 스킬 트리, 스테이지 선택 화면 등.

### 3. Layout Element 컴포넌트

Layout Group의 자식 요소에 `Layout Element` 컴포넌트를 추가하면, Layout Group의 전체적인 제어 규칙을 따르면서 **특정 자식만** 다른 크기 속성을 갖도록 예외를 둘 수 있습니다.

**Public Field (Layout Element)**

| 항목 | 설명 |
| :--- | :--- |
| `Ignore Layout` | `true`이면 이 요소는 Layout Group의 제어를 완전히 무시합니다. |
| `Min Width/Height` | 이 요소가 가질 수 있는 최소 너비/높이를 지정합니다. |
| `Preferred Width/Height` | Layout Group이 계산하는 기본 크기입니다. 이 값이 설정되면 Layout Group의 크기 제어를 무시하고 이 크기를 우선적으로 사용하려고 시도합니다. |
| `Flexible Width/Height` | 다른 요소들이 `Preferred` 크기를 모두 차지하고 남은 공간을 어떤 비율로 나눠 가질지를 결정합니다. 값이 클수록 더 많은 공간을 차지합니다. |

-   **활용 예시**: 채팅창에서 시스템 메시지만 다른 배경색과 함께 더 넓은 높이를 차지하도록 `Layout Element`의 `Preferred Height` 값을 더 크게 설정할 수 있습니다.

## 코드 예제

```csharp
// InventoryManager.cs
// Grid Layout Group으로 구성된 인벤토리에 동적으로 아이템을 추가하는 스크립트

using UnityEngine;
using UnityEngine.UI;

public class InventoryManager : MonoBehaviour
{
    // 인스펙터에서 아이템 슬롯 프리팹과, Grid Layout Group이 있는 부모 Panel을 할당
    public GameObject itemSlotPrefab;
    public Transform inventoryPanel; 

    public int numberOfItemsToAdd = 10;

    void Start()
    {
        // 게임 시작 시 10개의 아이템 슬롯을 인벤토리에 추가
        for (int i = 0; i < numberOfItemsToAdd; i++)
        {
            // 프리팹으로부터 새로운 슬롯 인스턴스를 생성
            GameObject newSlot = Instantiate(itemSlotPrefab);
            
            // 슬롯의 부모를 inventoryPanel로 설정
            // Instantiate의 두 번째 인자로 부모를 지정하거나, 아래처럼 SetParent를 사용
            newSlot.transform.SetParent(inventoryPanel, false); // worldPositionStays: false가 중요!

            // (옵션) 슬롯의 텍스트나 이미지를 설정
            // newSlot.GetComponentInChildren<TextMeshProUGUI>().text = "Item " + (i + 1);
        }
    }
}
```
*위 코드를 실행하면, `inventoryPanel`의 `Grid Layout Group`이 새로 추가된 `itemSlotPrefab`들을 자동으로 격자 정렬해줍니다.*

## 활용 예시

세로로 정렬되는 설정 메뉴를 만들어 봅시다.

1.  `UI -> Panel`을 생성하여 `SettingsMenu`라고 이름 짓습니다.
2.  `SettingsMenu`에 `Component -> Layout -> Vertical Layout Group`을 추가합니다.
3.  `Padding` 값을 `Left: 10, Right: 10, Top: 10, Bottom: 10`으로, `Spacing` 값을 `5`로 설정합니다.
4.  `Child Alignment`를 `UpperCenter`로, `Control Child Size`의 `Width`와 `Height`를 모두 체크합니다.
5.  `SettingsMenu`의 자식으로 `UI -> TextMeshPro`를 추가하여 "SOUND"라고 입력합니다.
6.  `SettingsMenu`의 자식으로 `UI -> Slider`를 추가합니다.
7.  `SettingsMenu`의 자식으로 `UI -> TextMeshPro`를 추가하여 "GRAPHICS"라고 입력합니다.
8.  `SettingsMenu`의 자식으로 `UI -> Toggle`을 추가합니다.
9.  게임을 실행하면 모든 자식 요소들이 `Vertical Layout Group`의 설정에 따라 위에서 아래로, 가운데 정렬되어 깔끔하게 배치된 것을 볼 수 있습니다.

## Note: 주의사항

-   **성능 문제 (Layout Rebuild)**: Layout Group의 자식 요소 중 하나라도 변경되면, 유니티는 전체 레이아웃을 다시 계산하는 'Layout Rebuild' 과정을 수행합니다. 자식 요소가 많고 변화가 잦은 UI(예: 실시간 순위표)에서는 이 과정이 성능 저하의 주된 원인이 될 수 있습니다. 이 경우, 자주 변하는 부분과 변하지 않는 부분을 다른 Layout Group으로 분리하는 등의 최적화가 필요합니다.
-   **수동 제어와의 충돌**: Layout Group에 의해 제어되는 자식의 `RectTransform` 값(위치, 크기 등)은 인스펙터에서 비활성화됩니다. 스크립트로 이 값을 직접 수정해도 Layout Group이 매 프레임 덮어쓰기 때문에 소용이 없습니다. 자식의 크기 등을 제어하고 싶다면 `Layout Element` 컴포넌트를 사용해야 합니다.

## Further Study: 확장 학습

-   **Content Size Fitter**: 자식 요소들의 전체 크기에 맞춰 부모 `RectTransform`의 크기를 자동으로 조절해주는 컴포넌트입니다. 내용의 길이에 따라 크기가 변하는 텍스트 창이나 목록 표시에 Layout Group과 함께 필수적으로 사용됩니다.
-   **Scroll View**: Layout Group으로 정렬된 콘텐츠가 화면을 벗어날 경우, 스크롤하여 볼 수 있게 해주는 복합 컴포넌트입니다.
-   **커스텀 레이아웃 그룹 (Custom Layout Group)**: `LayoutGroup` 클래스를 상속받아 원형이나 나선형 등 자신만의 독특한 정렬 규칙을 가진 커스텀 레이아웃 그룹을 만들 수 있습니다.
