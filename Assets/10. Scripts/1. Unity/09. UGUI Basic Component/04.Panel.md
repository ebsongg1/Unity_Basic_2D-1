# UI 요소의 그룹화, Panel

## Panel이란 무엇인가?

UI를 구성하다 보면 여러 버튼과 텍스트, 아이콘들을 하나의 그룹으로 묶어 관리하고 싶을 때가 많습니다. 예를 들어 '설정창'은 소리 조절 슬라이더, 그래픽 옵션 버튼, 나가기 버튼 등을 모두 포함하는 하나의 단위입니다. 이때 이 요소들을 담아두는 '쟁반'이나 '게시판' 같은 역할을 하는 것이 바로 **Panel(패널)** 입니다.

정의하자면, `Panel`은 다른 UI 요소들을 시각적으로나 계층적으로 그룹화하기 위해 사용되는 기본적인 UI 컨테이너입니다. 유니티에서 `UI > Panel` 메뉴로 생성하면, 반투명한 배경색을 가진 `Image` 컴포넌트가 부착된 `GameObject`가 만들어집니다. 즉, **Panel의 본질은 '배경'과 '그룹화' 역할을 하도록 미리 설정된 Image**라고 할 수 있습니다.

Panel은 그 자체로 특별한 기능을 가지기보다는, 다른 요소들을 담는 컨테이너로서의 역할이 핵심입니다. 설정 메뉴, 인벤토리 창, 팝업 알림 등 화면의 특정 구역을 차지하는 UI 그룹을 만들 때 가장 먼저 사용을 고려하게 되는 기본 중의 기본 요소입니다.

## 핵심 요약

-   Panel은 다른 UI 요소들을 **그룹화**하고 **배경**을 제공하기 위한 컨테이너입니다.
-   Panel의 실체는 `Image` 컴포넌트이므로, `Image`의 모든 속성(색상, 스프라이트, 타입 등)을 동일하게 가지고 있습니다.
-   주로 메뉴, 팝업창 등 UI의 특정 영역을 구분 짓는 용도로 사용됩니다.
-   Panel의 활성화/비활성화를 통해 자식으로 포함된 모든 UI 요소를 한 번에 제어할 수 있습니다.

## 세부 개념

Panel은 그 자체로 복잡한 개념을 가지지 않습니다. 핵심은 `Image` 컴포넌트를 배경으로 활용하고, `RectTransform`을 통해 자식 요소들을 담을 영역을 설정하는 것입니다.

### 1. Panel의 주요 역할

-   **그룹화 (Grouping)**: 관련된 UI 요소들(버튼, 텍스트 등)을 Panel의 자식으로 만들어 계층 구조를 정리합니다. 이렇게 하면 Panel의 위치를 옮길 때 모든 자식 요소가 함께 움직이며, `panel.SetActive(false)` 코드 한 줄로 그룹 전체를 숨기거나 보일 수 있어 관리가 매우 용이해집니다.
-   **배경 (Background)**: Panel의 `Image` 컴포넌트는 그룹의 배경 역할을 합니다.
    -   `Color` 속성을 조절하여 반투명한 어두운 배경(Dim 처리)을 만들거나, 특정 색상의 배경을 깔 수 있습니다.
    -   `Source Image`에 `Sliced` 타입의 스프라이트를 적용하여 멋진 테두리를 가진 창을 만들 수 있습니다.
-   **입력 차단 (Input Blocking)**: Panel의 `Image` 컴포넌트에 `Raycast Target`이 켜져 있으면, Panel은 자신보다 뒤에 있는 UI 요소나 3D 오브젝트에 대한 마우스 클릭을 막는 역할을 합니다. 팝업창이 떴을 때, 팝업창 외의 다른 부분은 클릭되지 않도록 하는 효과를 쉽게 구현할 수 있습니다.

**Public Field (Image 컴포넌트)**

| 항목 | 설명 |
| :--- | :--- |
| `Source Image` | Panel의 배경으로 사용할 `Sprite` 에셋을 지정합니다. |
| `Color` | 배경의 색상과 투명도를 조절합니다. |
| `Raycast Target` | `true`이면 Panel이 마우스 입력을 가로챕니다. 팝업창 뒤의 UI가 클릭되는 것을 막고 싶을 때 사용합니다. |

## 개념 다이어그램 & 코드 예제

```text
[Scene]
└── 📂 Canvas
    └── 🖼️ SettingsPanel (Panel)
        ├── 🔤 TitleText (TextMeshPro)
        ├── 🔘 SoundToggleButton (Button)
        ├── 🎚️ VolumeSlider (Slider)
        └── 🔘 CloseButton (Button)
```

*설정창(SettingsPanel)이라는 하나의 단위 안에 제목, 버튼, 슬라이더가 모두 포함되어 함께 관리됩니다.* 

```csharp
// SettingsPanelController.cs
// 설정 패널을 열고 닫는 기능을 제어하는 스크립트

using UnityEngine;

public class SettingsPanelController : MonoBehaviour
{
    // 인스펙터에서 제어할 설정 패널 GameObject를 할당
    public GameObject settingsPanel;

    void Start()
    {
        // 게임 시작 시에는 설정 패널을 비활성화
        if (settingsPanel != null)
        {
            settingsPanel.SetActive(false);
        }
    }

    // 이 함수를 '설정 열기' 버튼의 OnClick 이벤트에 연결
    public void OpenPanel()
    {
        if (settingsPanel != null)
        {
            settingsPanel.SetActive(true);
        }
    }

    // 이 함수를 '닫기' 버튼의 OnClick 이벤트에 연결
    public void ClosePanel()
    {
        if (settingsPanel != null)
        {
            settingsPanel.SetActive(false);
        }
    }

    // Update에서 키 입력으로 패널을 토글할 수도 있음
    void Update()
    {
        if (Input.GetKeyDown(KeyCode.Escape))
        {
            // 패널의 현재 활성화 상태를 뒤집음
            if (settingsPanel != null)
            {
                bool isActive = settingsPanel.activeSelf;
                settingsPanel.SetActive(!isActive);
            }
        }
    }
}
```

## 활용 예시

게임 오버 화면을 Panel을 이용해 만들어 봅시다.

1.  `Hierarchy` 창에서 `UI -> Panel`을 선택하여 `GameOverPanel`을 생성합니다.
2.  `GameOverPanel`의 `Image` 컴포넌트의 `Color`를 약간 어두운 붉은색, 반투명하게 조절하여 게임 오버 분위기를 연출합니다.
3.  `GameOverPanel`의 자식으로 `UI -> Text - TextMeshPro`를 추가하여 "GAME OVER" 텍스트를 표시합니다. 텍스트의 폰트, 크기, 색상을 적절히 조절합니다.
4.  `GameOverPanel`의 자식으로 `UI -> Button - TextMeshPro`를 2개 추가하여 '다시 시작' 버튼과 '메인 메뉴로' 버튼을 만듭니다.
5.  평소에는 `GameOverPanel`을 비활성화(`SetActive(false)`) 해 둡니다.
6.  플레이어의 체력이 0이 되는 순간, 게임 매니저 스크립트에서 `gameOverPanel.SetActive(true)` 코드를 호출하여 패널 전체를 화면에 표시합니다.

## Note: 주의사항

-   **Panel vs Image**: `UI -> Panel`로 생성하는 것과, `UI -> Image`로 생성한 뒤 옵션을 조절하는 것은 기능적으로 아무런 차이가 없습니다. Panel 메뉴는 단지 배경 및 컨테이너로 자주 사용되는 `Image` 컴포넌트의 설정값을 미리 세팅해 둔 '프리셋' 또는 '템플릿'일 뿐입니다.
-   **성능**: 불필요한 Panel 사용은 `Draw Call`을 늘릴 수 있습니다. 단순히 계층 구조상의 그룹화가 목적이라면, `Image` 컴포넌트가 없는 빈 `GameObject`를 사용하는 것이 렌더링 부하가 없어 더 효율적입니다. 시각적인 배경이 필요할 때만 Panel(또는 Image)을 사용하세요.

## Further Study: 확장 학습

-   **스크롤 뷰 (Scroll View)**: Panel과 유사하지만, 담고 있는 내용이 Panel의 크기를 벗어날 때 스크롤하여 볼 수 있는 기능을 제공하는 컴포넌트입니다. 내용이 많은 인벤토리나 상점 목록을 만들 때 필수적입니다.
-   **Layout Groups**: `Vertical Layout Group`, `Horizontal Layout Group`, `Grid Layout Group` 등 Panel과 함께 사용되어 자식 UI 요소들을 자동으로 정렬해주는 강력한 기능들에 대해 학습해보세요.
