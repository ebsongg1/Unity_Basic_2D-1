# 값의 조절과 표현, Slider

## Slider란 무엇인가?

오디오의 볼륨 조절기나, 영상 플레이어의 재생 위치를 조절하는 탐색 바를 떠올려봅시다. 이들은 모두 사용자가 손잡이(Handle)를 움직여 특정 범위 내의 값을 선택하거나 현재 상태를 시각적으로 확인하는 데 사용됩니다. UI에서 **Slider(슬라이더)** 는 바로 이러한 '연속적인 값의 조절 및 표현'을 위한 컴포넌트입니다.

정의하자면, `Slider`는 사용자가 드래그(Drag)를 통해 최소값과 최대값 사이의 특정 수치(Value)를 선택할 수 있게 해주는 상호작용 컴포넌트입니다. 주로 게임 설정 메뉴의 볼륨 조절, 밝기 조절 등에 사용되며, 플레이어의 체력(HP)이나 경험치(EXP) 바처럼 현재 상태를 시각적으로 보여주는 프로그레스 바(Progress Bar)의 역할도 수행할 수 있습니다.

`Slider`는 보통 배경이 되는 `Background` 이미지, 채워지는 영역을 나타내는 `Fill` 이미지, 그리고 사용자가 직접 움직이는 `Handle` 이미지의 조합으로 구성됩니다.

## 핵심 요약

-   Slider는 사용자가 드래그를 통해 **특정 범위 내의 숫자 값**을 선택하거나 확인할 수 있는 컴포넌트입니다.
-   **Min Value**와 **Max Value**로 값의 범위를 설정하고, **Value** 속성으로 현재 값을 가져오거나 변경합니다.
-   **On Value Changed()** 이벤트를 통해 슬라이더의 값이 변경될 때마다 특정 함수를 실시간으로 호출할 수 있습니다.
-   게임 설정(볼륨, 밝기)과 상태 표시(체력 바, 경험치 바) 등 다용도로 활용됩니다.

## 세부 개념

### 1. Slider 컴포넌트의 구조와 주요 속성

`Slider`를 생성하면 여러 개의 `GameObject`가 계층적으로 만들어집니다. 각 부분의 역할과 주요 속성을 이해하는 것이 중요합니다.

-   **Slider (Root)**: `Slider` 컴포넌트가 있는 루트 오브젝트입니다.
    -   **Background**: 슬라이더의 전체 영역을 나타내는 배경 이미지입니다.
    -   **Fill Area**: 채워지는 영역(`Fill`)을 담는 컨테이너입니다.
        -   **Fill**: 슬라이더의 현재 값을 시각적으로 보여주는 채워진 이미지입니다. `Value`에 따라 크기가 변합니다.
    -   **Handle Slide Area**: 핸들이 움직일 수 있는 영역을 정의합니다.
        -   **Handle**: 사용자가 직접 드래그하는 손잡이 이미지입니다.

**Public Field (Slider 컴포넌트)**

| 항목 | 설명 |
| :--- | :--- |
| `Interactable` | 슬라이더를 사용자가 조작할 수 있는지 여부를 결정합니다. `false`이면 조작이 불가능하고 비활성화된 모습으로 보입니다. |
| `Fill Rect` | 채워지는 영역을 담당할 `RectTransform`을 지정합니다. 보통 `Fill` 오브젝트가 연결됩니다. |
| `Handle Rect` | 손잡이 역할을 할 `RectTransform`을 지정합니다. 보통 `Handle` 오브젝트가 연결됩니다. |
| `Direction` | 슬라이더의 값이 증가하는 방향을 결정합니다. `Left To Right`, `Right To Left`, `Bottom To Top`, `Top To Bottom` 중에서 선택할 수 있습니다. |
| `Min Value` | 슬라이더가 가질 수 있는 최소값입니다. |
| `Max Value` | 슬라이더가 가질 수 있는 최대값입니다. |
| `Whole Numbers` | `true`로 설정하면 `Value`가 정수 값으로만 변경됩니다. (예: 1, 2, 3...) |
| `Value` | 슬라이더의 현재 값입니다. 스크립트에서 이 값을 읽거나 설정하여 슬라이더를 제어합니다. |

### 2. On Value Changed (Single) 이벤트

`Slider`의 가장 중요한 이벤트입니다. 사용자가 핸들을 드래그하여 `Value`가 변경될 때마다 **실시간으로** 호출됩니다.

-   `float` 타입의 매개변수를 받는 함수를 연결할 수 있으며, 이 매개변수에는 변경된 `Value` 값이 자동으로 전달됩니다.
-   이를 통해 슬라이더의 값 변화를 즉시 다른 시스템에 반영할 수 있습니다. (예: 슬라이더를 움직이는 즉시 게임 볼륨이 바뀌는 효과)

## 개념 다이어그램 & 코드 예제

```
[Scene]
└── 📂 Canvas
    └── 🎚️ VolumeSlider (Slider)
        ├── 🖼️ Background (Image)
        ├── 📂 Fill Area
        │   └── 🖼️ Fill (Image)
        └── 📂 Handle Slide Area
            └── 🖼️ Handle (Image)
```
*슬라이더는 배경, 채우기, 핸들 등 여러 이미지의 조합으로 이루어져 있습니다.* 

```csharp
// SoundManager.cs
// 슬라이더를 이용해 게임의 전체 볼륨을 조절하는 스크립트

using UnityEngine;
using UnityEngine.UI; // Slider를 사용하기 위해 필요
using UnityEngine.Audio; // AudioMixer를 제어하기 위해 필요

public class SoundManager : MonoBehaviour
{
    public AudioMixer masterMixer;
    public Slider volumeSlider;

    void Start()
    {
        // 슬라이더의 OnValueChanged 이벤트에 리스너를 코드로 등록
        if (volumeSlider != null)
        {
            // 슬라이더 값이 변할 때마다 SetMasterVolume 함수가 호출되도록 설정
            volumeSlider.onValueChanged.AddListener(SetMasterVolume);

            // 초기 볼륨값으로 슬라이더 상태를 초기화
            float initialVolume;
            masterMixer.GetFloat("MasterVolume", out initialVolume);
            volumeSlider.value = Mathf.Pow(10, initialVolume / 20);
        }
    }

    // 이 함수를 슬라이더의 OnValueChanged 이벤트에 연결 (인스펙터 또는 코드)
    // 매개변수 value에는 슬라이더의 현재 값이 자동으로 전달됨 (0.0 ~ 1.0)
    public void SetMasterVolume(float value)
    {
        if (value <= 0.0001f)
        {
            // 볼륨이 0에 가까우면 완전히 음소거 (-80dB)
            masterMixer.SetFloat("MasterVolume", -80);
        }
        else
        {
            // 슬라이더 값(선형)을 데시벨(로그) 스케일로 변환하여 AudioMixer에 적용
            // Log10(value) * 20 이 데시벨 변환 공식
            masterMixer.SetFloat("MasterVolume", Mathf.Log10(value) * 20);
        }
    }
}
```

## 활용 예시

플레이어의 체력 바(HP Bar)를 슬라이더로 만들어 봅시다. 이 체력 바는 값 표시에만 사용되므로 사용자가 직접 조작할 수는 없습니다.

1.  `Hierarchy`에서 `UI -> Slider`를 생성합니다.
2.  생성된 `Slider`의 자식 오브젝트인 `Handle Slide Area`를 삭제하거나 비활성화합니다. 사용자가 조작할 필요가 없기 때문입니다.
3.  `Slider` 컴포넌트의 `Interactable` 속성을 `false`로 설정합니다.
4.  `Fill` 오브젝트의 `Image` 컴포넌트 `Color`를 체력 바에 어울리는 붉은색으로 변경합니다.
5.  `Slider`의 `Min Value`를 0으로, `Max Value`를 플레이어의 최대 체력(예: 100)으로 설정합니다.
6.  플레이어를 관리하는 스크립트(예: `PlayerHealth.cs`)에서 `Slider`를 `public` 변수로 참조합니다.
7.  플레이어가 피해를 입거나 회복할 때마다, 스크립트에서 `hpSlider.value = currentPlayerHealth;` 와 같이 코드를 작성하여 슬라이더의 값을 현재 체력과 동기화시켜줍니다.

## Note: 주의사항

-   **값의 범위**: `Slider`의 `Value`는 항상 `Min Value`와 `Max Value` 사이의 값으로 제한됩니다. 스크립트에서 이 범위를 벗어나는 값을 설정해도 자동으로 범위 내의 값으로 보정됩니다.
-   **정수 값 사용**: `Whole Numbers`를 체크하면 슬라이더가 정수 단위로만 움직입니다. '아이템 개수 선택'처럼 소수점이 필요 없는 값을 조절할 때 유용합니다.
-   **로그 스케일 변환**: 사운드 볼륨처럼 사람이 인식하는 크기가 선형적이지 않은 값을 제어할 때는, 슬라이더의 선형적인 값(0~1)을 로그 스케일(데시벨 등)로 변환하여 적용해야 훨씬 자연스러운 조작감을 제공할 수 있습니다. (코드 예제 참고)

## Further Study: 확장 학습

-   **Audio Mixer**: 유니티의 `Audio Mixer`를 사용하면 BGM, SFX, Voice 등 오디오 소스를 그룹별로 나누고, 각 그룹의 볼륨이나 효과를 통합적으로 관리할 수 있습니다. 슬라이더와 연동하여 강력한 사운드 설정 기능을 만들 수 있습니다.
-   **프로그레스 바 (Progress Bar)**: `Slider`는 프로그레스 바를 만드는 가장 쉬운 방법 중 하나입니다. 로딩 진행률, 경험치 바, 캐스팅 바 등 시간에 따라 변하는 상태를 시각적으로 표현하는 다양한 방법에 대해 연구해보세요.
-   **Input Field**: 슬라이더가 연속적인 값을 다룬다면, `Input Field` 컴포넌트는 사용자가 직접 텍스트(문자, 숫자 등)를 키보드로 입력하게 할 수 있는 컴포넌트입니다. 플레이어 이름 입력, 채팅창 등에 사용됩니다.
