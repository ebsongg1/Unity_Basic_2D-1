# 거대한 콘텐츠를 탐색하는 창, Scroll View

## Scroll View란 무엇인가?

긴 웹 페이지를 보거나, 스마트폰에서 수많은 사진 목록을 훑어볼 때 우리는 손가락으로 화면을 밀어(scroll) 보이지 않는 부분을 탐색합니다. 즉, '제한된 크기의 창(화면)'을 통해 '거대한 크기의 콘텐츠'를 들여다보는 것입니다.

UI에서 **Scroll View(스크롤 뷰)** 는 바로 이러한 '스크롤' 기능을 구현해놓은 복합적인 컴포넌트입니다. `Layout Group`으로 정렬된 아이템 목록이 너무 길어져서 화면을 벗어날 때, 사용자가 마우스나 터치로 이 목록을 위아래 또는 좌우로 움직여서 모든 콘텐츠를 볼 수 있게 해줍니다.

정의하자면, `Scroll View`는 제한된 뷰(View) 영역을 통해 그보다 큰 콘텐츠 영역을 스크롤하며 볼 수 있도록 해주는 UGUI의 핵심 컨테이너 컴포넌트입니다. 아이템이 수백 개에 달하는 인벤토리, 끝없이 이어지는 상점 목록, 긴 설명글 등 많은 양의 정보를 표시해야 할 때 필수적으로 사용됩니다.

## 핵심 요약

-   Scroll View는 **화면보다 큰 콘텐츠를 스크롤**하여 볼 수 있게 해주는 복합 컴포넌트입니다.
-   **Viewport(뷰포트)** 라는 창을 통해 **Content(콘텐츠)** 영역을 들여다보는 구조로 이루어져 있습니다.
-   **Scrollbar(스크롤바)** 를 통해 현재 위치를 시각적으로 표시하고 조작할 수 있습니다.
-   주로 `Layout Group` 및 `Content Size Fitter`와 함께 사용되어 동적 목록을 표시합니다.

## 세부 개념

### 1. Scroll View의 구조

`Scroll View`를 생성하면 여러 컴포넌트가 조합된 복잡한 계층 구조가 만들어집니다. 각 부분의 역할을 이해하는 것이 매우 중요합니다.

-   **Scroll View (Root)**: `Scroll Rect` 컴포넌트가 있는 루트 오브젝트.
    -   **Viewport**: '창문'의 역할을 하는 영역입니다. 실제 사용자에게 보이는 부분이며, 보통 `Mask` 컴포넌트가 있어 이 영역을 벗어나는 콘텐츠를 가려줍니다.
        -   **Content**: 실제 콘텐츠들이 위치하는 패널입니다. `Viewport`의 자식이며, `Viewport`보다 크기가 훨씬 클 수 있습니다. 이 `Content` 패널에 `Vertical/Grid Layout Group`과 `Content Size Fitter`를 추가하여 자식 아이템들을 정렬하고 크기를 자동으로 조절합니다.
    -   **Scrollbar Horizontal**: 가로 스크롤바.
    -   **Scrollbar Vertical**: 세로 스크롤바.

### 2. Scroll Rect 컴포넌트의 주요 속성

`Scroll View`의 핵심 로직은 루트 오브젝트의 `Scroll Rect` 컴포넌트가 담당합니다.

**Public Field (Scroll Rect)**

| 항목 | 설명 |
| :--- | :--- |
| `Content` | 스크롤될 콘텐츠 영역의 `RectTransform`을 지정합니다. (위 구조의 `Content` 패널) |
| `Horizontal` | 수평 스크롤을 활성화할지 여부를 결정합니다. |
| `Vertical` | 수직 스크롤을 활성화할지 여부를 결정합니다. |
| `Movement Type` | 스크롤 경계에 도달했을 때의 동작을 결정합니다. `Unrestricted`(제한 없음), `Elastic`(탄성 효과), `Clamped`(경계에서 멈춤) |
| `Inertia` | `true`이면 사용자가 스크롤 후 손을 떼도 관성이 적용되어 잠시 더 미끄러집니다. `Deceleration Rate`로 감속도를 조절합니다. |
| `Horizontal/Vertical Scrollbar` | 스크롤과 연동될 스크롤바를 지정합니다. |
| `On Value Changed (Vector2)` | 스크롤 위치가 변경될 때마다 실시간으로 호출되는 이벤트입니다. `Vector2` 값은 정규화된 현재 스크롤 위치(x, y 각각 0~1)를 나타냅니다. |

### 3. 동작 원리

1.  `Content` 패널에 `Vertical Layout Group`과 `Content Size Fitter` (Vertical Fit: PreferredSize)를 추가합니다.
2.  `Content` 패널의 자식으로 아이템(버튼, 이미지 등)들을 추가합니다.
3.  `Vertical Layout Group`이 아이템들을 세로로 정렬하고, `Content Size Fitter`가 모든 아이템을 포함하는 총 높이를 계산하여 `Content` 패널의 높이를 늘립니다.
4.  `Content`의 높이가 `Viewport`의 높이보다 커지면, `Scroll Rect`는 자동으로 스크롤이 가능하도록 활성화하고 스크롤바를 표시합니다.
5.  사용자가 `Viewport` 영역을 드래그하면, `Scroll Rect`는 `Content` 패널의 `anchoredPosition`을 변경하여 스크롤 효과를 만들어냅니다.

## 코드 예제

```csharp
// ScrollToTop.cs
// 버튼을 누르면 스크롤 뷰를 맨 위 또는 맨 아래로 이동시키는 스크립트

using UnityEngine;
using UnityEngine.UI;

public class ScrollToTop : MonoBehaviour
{
    public ScrollRect myScrollRect;

    // 스크롤 뷰를 맨 위로 이동시키는 함수
    public void MoveToTop()
    {
        if (myScrollRect != null)
        {
            // verticalNormalizedPosition = 1 이 맨 위를 의미
            myScrollRect.verticalNormalizedPosition = 1f;
        }
    }

    // 스크롤 뷰를 맨 아래로 이동시키는 함수
    public void MoveToBottom()
    {
        if (myScrollRect != null)
        {
            // verticalNormalizedPosition = 0 이 맨 아래를 의미
            myScrollRect.verticalNormalizedPosition = 0f;
        }
    }

    void Start()
    {
        // 스크롤 위치가 변할 때마다 로그를 출력하는 리스너 등록
        myScrollRect.onValueChanged.AddListener(OnScroll);
    }

    private void OnScroll(Vector2 position)
    {
        // position.y 값은 위(1) ~ 아래(0) 사이의 값을 가짐
        Debug.Log("Current Scroll Position: " + position.y);
    }
}
```

## 활용 예시

수직으로 스크롤되는 간단한 아이템 목록을 만들어 봅시다.

1.  `Hierarchy`에서 `UI -> Scroll View`를 생성합니다.
2.  `Scroll View`의 구조를 확인하고, `Content` 오브젝트(`Scroll View -> Viewport -> Content`)를 선택합니다.
3.  `Content`에 `Component -> Layout -> Vertical Layout Group`을 추가합니다. `Spacing`을 10 정도로 설정하고, `Child Control Size`의 `Width`를 체크합니다.
4.  `Content`에 `Component -> Layout -> Content Size Fitter`를 추가하고, `Vertical Fit`을 `PreferredSize`로 설정합니다.
5.  아이템으로 사용할 프리팹(예: `Button` 또는 `Image`)을 만듭니다. 이 프리팹에 `Layout Element` 컴포넌트를 추가하고 `Preferred Height`를 100 정도로 설정합니다.
6.  스크립트나 수동으로, 이 아이템 프리팹을 20개 정도 `Content`의 자식으로 추가(`Instantiate`)합니다.
7.  게임을 실행하면, `Content`의 높이가 아이템들의 총 높이에 맞춰 늘어나고, `Scroll View`에서 마우스 휠이나 드래그를 통해 목록을 스크롤할 수 있게 됩니다.

## Note: 주의사항

-   **성능 최적화 (가상화)**: `Scroll View`에 수백, 수천 개의 아이템을 한 번에 생성하면 엄청난 성능 저하가 발생합니다. 이 경우, 화면에 **보이는 영역의 아이템만 실시간으로 생성하고 재활용**하는 '가상화(Virtualization)' 또는 '리사이클러 뷰(Recycler View)' 기법이 필수적입니다. 이는 직접 구현하거나 에셋 스토어의 전문적인 UI 솔루션을 사용해야 합니다.
-   **`Content`의 앵커와 피벗**: `Scroll View`가 정상적으로 작동하려면 `Content`의 `RectTransform` 앵커와 피벗 설정이 중요합니다. 일반적으로 수직 스크롤의 경우, `Content`의 피벗 Y값을 1(상단 기준)로, 앵커 Min Y를 0, Max Y를 1로 설정하여 `Viewport`의 상단에 정렬되도록 합니다.
-   **스크롤바 숨기기**: 스크롤바가 필요 없다면 `Scrollbar Horizontal`, `Scrollbar Vertical` 오브젝트를 비활성화하거나 삭제하면 됩니다. `Scroll Rect`의 스크롤바 참조 필드도 `None`으로 설정해줍니다.

## Further Study: 확장 학습

-   **가상 스크롤 뷰 (Virtual Scroll View)**: 대규모 데이터를 효율적으로 처리하기 위한 가상화 기법의 원리를 학습하고, 간단한 버전을 직접 구현해보세요. `onValueChanged` 이벤트를 통해 현재 스크롤 위치를 파악하고, 그에 맞는 데이터만 활성화하는 것이 핵심입니다.
-   **페이지 스크롤 (Paged Scroll)**: 스크롤이 부드럽게 이어지는 것이 아니라, 책장을 넘기듯 페이지 단위로 딱딱 끊어지게 만드는 기법입니다. `Scroll Rect`의 `Inertia`를 끄고, 스크롤이 끝났을 때 가장 가까운 페이지로 위치를 보정해주는 스크립트를 작성하여 구현할 수 있습니다.
-   **Nested Scroll View (중첩 스크롤 뷰)**: `Scroll View` 안에 또 다른 `Scroll View`를 넣는 복잡한 구조입니다. (예: 세로로 스크롤되는 상점 목록 안에, 각 아이템이 가로로 스크롤되는 경우) 이 경우, 부모와 자식 스크롤 뷰의 입력 이벤트를 제대로 분리하여 처리해주지 않으면 충돌이 발생하므로 고급 이벤트 처리 기법이 필요합니다.
