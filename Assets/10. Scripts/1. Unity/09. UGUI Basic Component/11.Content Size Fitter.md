# 내용에 따라 크기가 변하는, Content Size Fitter

## Content Size Fitter란?

풍선에 바람을 불어넣는 것을 생각해봅시다. 바람을 조금 불면 풍선은 작고, 많이 불면 내용물(공기)에 맞춰 풍선이 커집니다. 즉, 내용물의 양에 따라 컨테이너의 크기가 결정됩니다.

UI에서 **Content Size Fitter(콘텐츠 사이즈 피터)** 는 바로 이 '내용물에 맞춰 크기가 변하는 풍선'과 같은 역할을 합니다. 이 컴포넌트를 `RectTransform`이 있는 UI 요소에 추가하면, 그 UI 요소는 자신의 자식 요소들이나 `Layout Element`가 차지하는 공간의 크기에 맞춰 자신의 너비나 높이를 자동으로 조절하게 됩니다.

정의하자면, `Content Size Fitter`는 자식 레이아웃 요소들의 최소(Min) 또는 선호(Preferred) 크기를 기반으로 자신의 `RectTransform` 크기를 맞추는 컴포넌트입니다. 텍스트 길이에 따라 배경의 크기가 변하는 채팅 말풍선, 아이템 개수에 따라 목록의 높이가 변하는 인벤토리 창 등을 만들 때 매우 유용합니다.

## 핵심 요약

-   Content Size Fitter는 **자식 콘텐츠의 크기에 맞춰 부모의 크기를 자동으로 조절**하는 컴포넌트입니다.
-   **Horizontal Fit**과 **Vertical Fit** 속성을 통해 너비와 높이를 각각 어떻게 조절할지 결정합니다. (`Unconstrained`, `MinSize`, `PreferredSize`)
-   주로 `Layout Group`과 함께 사용되어, 자식들의 총합 크기를 계산하고 그에 맞춰 부모 패널의 크기를 조절하는 역할을 합니다.
-   동적으로 내용이 변하는 UI를 만들 때 필수적입니다.

## 세부 개념

### Fit Mode (피팅 모드)

`Content Size Fitter`는 수평(Horizontal)과 수직(Vertical) 방향에 대해 독립적으로 크기 조절 방식을 설정할 수 있습니다.

**Public Field**

| 항목 | 설명 |
| :--- | :--- |
| `Horizontal Fit` | 너비를 어떻게 조절할지 결정합니다. |
| `Vertical Fit` | 높이를 어떻게 조절할지 결정합니다. |

각 Fit 속성은 다음 세 가지 모드 중 하나를 선택할 수 있습니다.

-   **Unconstrained**: 크기를 조절하지 않습니다. 해당 방향으로는 `Content Size Fitter`가 작동하지 않는 것과 같습니다.
-   **MinSize**: 모든 자식 `Layout Element`들의 `Min Size`를 합한 크기에 맞춰 자신의 크기를 조절합니다.
-   **PreferredSize**: 모든 자식 `Layout Element`들의 `Preferred Size`를 합한 크기에 맞춰 자신의 크기를 조절합니다. 이것이 가장 일반적으로 사용되는 모드입니다.

**어떻게 크기를 계산하는가?**
`Content Size Fitter`는 자식 요소들에게 부착된 `ILayoutElement` 인터페이스를 구현한 모든 컴포넌트(예: `Layout Element`, `Image`, `Text`)를 검색하여 그들의 `preferredWidth/Height` 값을 가져옵니다. `Layout Group`이 있다면, `Padding`과 `Spacing` 값까지 모두 더하여 최종적인 '선호 크기'를 계산하고, 자신의 `RectTransform` 크기를 그 값으로 설정합니다.

-   **잘못된 사용 예시**: `Content Size Fitter`와 `Layout Group`을 **동일한** 게임 오브젝트에 함께 배치하는 경우.
    -   **문제점**: `Content Size Fitter`는 자식 크기에 따라 부모 크기를 조절하려 하고, `Layout Group`은 부모 크기에 따라 자식 크기를 조절하려 하여 서로 충돌하고 무한 루프에 빠질 수 있습니다.
    -   **올바른 구조**: 자식들을 담는 **콘텐츠 패널**과, 그 콘텐츠 패널을 담는 **뷰포트(Viewport)** 를 분리해야 합니다.
        -   `Viewport` (부모): `Image`, `Mask` 컴포넌트. 고정된 크기를 가집니다.
        -   `Content` (자식): `Vertical/Horizontal Layout Group`, `Content Size Fitter` 컴포넌트. `Viewport`의 자식으로 있으며, 내용에 따라 크기가 변합니다.

## 코드 예제

`Content Size Fitter`는 보통 다른 레이아웃 컴포넌트와 함께 작동하므로, 특정 기능을 위한 단독 코드 예제보다는 조합을 통해 그 기능을 확인하는 경우가 많습니다. 아래는 `Layout Group`과 함께 동적으로 UI를 생성하는 예시입니다.

```csharp
// ChatManager.cs
// 새로운 메시지가 추가될 때마다 Content Size Fitter가 적용된 패널의 높이가 늘어나는 예제

using UnityEngine;
using UnityEngine.UI;
using TMPro; // TextMeshPro 사용 시

public class ChatManager : MonoBehaviour
{
    // 인스펙터에서 할당
    public TMP_InputField chatInputField;
    public GameObject chatMessagePrefab; // 텍스트 컴포넌트가 있는 프리팹
    
    // Vertical Layout Group과 Content Size Fitter가 있는 패널
    public RectTransform contentPanel; 

    void Start()
    {
        chatInputField.onSubmit.AddListener(AddNewMessage);
    }

    private void AddNewMessage(string message)
    {
        if (string.IsNullOrWhiteSpace(message)) return;

        // 새로운 메시지 오브젝트 생성
        GameObject newMessageObj = Instantiate(chatMessagePrefab, contentPanel);
        
        // 메시지 텍스트 설정
        TMP_Text messageText = newMessageObj.GetComponentInChildren<TMP_Text>();
        messageText.text = message;

        // 입력 필드 초기화
        chatInputField.text = "";
        chatInputField.ActivateInputField(); // 다시 입력 필드에 포커스

        // 여기! 새로운 자식이 추가되면, contentPanel의 Vertical Layout Group이
        // 자식들을 정렬하고, Content Size Fitter가 그 총 높이에 맞춰
        // contentPanel의 높이를 자동으로 늘려줍니다.
    }
}
```

## 활용 예시

텍스트 길이에 따라 배경 이미지의 크기가 자동으로 변하는 버튼을 만들어 봅시다.

1.  `UI -> Button - TextMeshPro`를 생성하고 이름을 `DynamicButton`으로 변경합니다.
2.  `DynamicButton`의 자식인 `Text (TMP)` 오브젝트를 선택하고, `RectTransform`의 앵커 프리셋을 `stretch-stretch`(가운데 꽉 채우기)로 설정합니다.
3.  `DynamicButton` 오브젝트를 선택하고, `Component -> Layout -> Horizontal Layout Group`을 추가합니다. `Padding`을 적절히 조절하여 텍스트와 버튼 테두리 사이에 여백을 줍니다.
4.  `DynamicButton` 오브젝트에 `Component -> Layout -> Content Size Fitter`를 추가합니다.
5.  `Content Size Fitter`의 `Horizontal Fit`을 `PreferredSize`로 설정합니다.
6.  이제 `Text (TMP)` 오브젝트의 `Text` 내용을 인스펙터에서 변경해봅니다. "짧은 텍스트"를 입력했을 때와 "아주 아주 긴 텍스트를 입력했을 때" 버튼의 너비가 텍스트 길이에 맞춰 자동으로 변하는 것을 볼 수 있습니다.
7.  **중요**: 버튼 배경이 `Sliced` 타입 이미지여야 찌그러지지 않고 예쁘게 늘어납니다.

## Note: 주의사항

-   **앵커 설정의 중요성**: `Content Size Fitter`는 `RectTransform`의 앵커 설정에 크게 영향을 받습니다. 일반적으로 `Content Size Fitter`를 사용하는 UI 요소는 앵커를 한 점에 모아두는 것이(예: `middle-center` 또는 `top-left`) 예상치 못한 크기 변화를 막는 데 도움이 됩니다.
-   **Layout Group과의 충돌**: 앞서 설명했듯이, `Layout Group`과 `Content Size Fitter`는 동일한 `GameObject`에 있으면 안 됩니다. `Scroll View`의 `Viewport`-`Content` 구조처럼 부모-자식 관계로 분리하여 사용해야 합니다.
-   **성능**: `Content Size Fitter` 역시 `Layout Group`처럼 레이아웃 재계산(Layout Rebuild)을 유발하므로, 너무 많은 요소에 사용하거나 매 프레임 내용이 변하는 UI에 적용할 경우 성능을 주의 깊게 살펴봐야 합니다.

## Further Study: 확장 학습

-   **Scroll View**: `Content Size Fitter`와 `Layout Group`의 가장 대표적인 활용 사례입니다. 내용이 화면을 벗어날 정도로 커졌을 때 스크롤 기능을 제공하는 `Scroll View`의 내부 구조를 분석해보세요.
-   **ILayoutElement 인터페이스**: `Text`, `Image`, `Layout Element` 컴포넌트가 어떻게 `Content Size Fitter`와 `Layout Group`에 자신의 크기 정보를 제공하는지 `ILayoutElement` 인터페이스를 통해 학습해보세요. 이를 통해 자신만의 커스텀 UI 컴포넌트가 레이아웃 시스템과 상호작용하도록 만들 수 있습니다.
