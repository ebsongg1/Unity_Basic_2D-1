# UI의 시각적 표현, Image

## Image란?

`Canvas`가 화가의 캔버스이고, `RectTransform`이 그림을 그릴 영역의 크기와 위치를 정하는 도구라면, **Image(이미지)** 컴포넌트는 그 영역 위에 실제 그림을 그려 넣는 '물감'과 같습니다. 아이콘, 캐릭터 초상화, 배경 그림 등 UI에 표시되는 거의 모든 시각적 그래픽은 `Image` 컴포넌트를 통해 구현됩니다.

정의하자면, `Image`는 `RectTransform`으로 정의된 사각 영역 안에 **스프라이트(Sprite)** 를 표시하는 UGUI의 가장 기본적인 그래픽 컴포넌트입니다. 텍스트가 아닌 모든 종류의 그래픽을 UI에 표현하기 위한 첫걸음이라고 할 수 있습니다.

`Image` 컴포넌트가 필요한 이유는 명확합니다. 버튼은 아이콘이 있어야 알아보기 쉽고, 체력 바는 색깔이 채워진 막대가 있어야 상태를 알 수 있으며, 메뉴는 배경 이미지가 있어야 보기 좋습니다. `Image` 컴포넌트는 단순히 이미지를 보여주는 것을 넘어, 늘리기, 타일처럼 반복하기, 특정 방향으로 채우기 등 동적이고 매력적인 UI를 만드는 데 필수적인 다양한 표시 방식을 제공합니다.

## 핵심 요약

-   `Image` 컴포넌트는 UI에 **스프라이트(Sprite)** 를 표시하는 가장 기본적인 그래픽 컴포넌트입니다.
-   **Source Image** 필드에 표시할 `Sprite` 에셋을 할당하여 사용합니다.
-   **Image Type** 속성을 통해 스프라이트를 표시하는 다양한 방식(Simple, Sliced, Tiled, Filled)을 선택할 수 있습니다.
-   **Raycast Target** 옵션은 이 이미지가 마우스나 터치 입력을 감지할지(가로챌지) 여부를 결정하는 중요한 성능 관련 옵션입니다.

## 세부 개념

### Image Type (이미지 타입)

`Image Type`은 `Source Image`로 지정된 스프라이트를 `RectTransform` 영역에 어떻게 그려낼지를 결정하는 가장 핵심적인 속성입니다. 각 타입의 특징을 이해하면 훨씬 다채롭고 효율적인 UI를 만들 수 있습니다.

#### 1) Simple

가장 기본적인 모드입니다. 지정된 스프라이트 전체를 `RectTransform` 크기에 맞춰 스케일링합니다.

**Public Field**

| 항목 | 설명 |
| :--- | :--- |
| `Source Image` | 표시할 `Sprite` 에셋을 지정합니다. |
| `Color` | 스프라이트에 곱해지는 색상입니다. 흰색이 원본 색상이며, 다른 색을 섞거나 알파(Alpha)값으로 투명도를 조절할 수 있습니다. |
| `Material` | 이미지를 렌더링할 때 사용할 머티리얼을 지정합니다. 특수한 셰이더 효과를 적용할 때 사용합니다. |
| `Raycast Target` | `true`이면 이 이미지가 그래픽 레이캐스트를 막습니다. 즉, 마우스 클릭 등의 입력을 감지합니다. `false`이면 입력이 이 이미지를 통과하여 뒤에 있는 요소로 전달됩니다. |
| `Preserve Aspect` | `true`이면 `RectTransform`의 크기가 변해도 스프라이트의 원본 가로세로 비율을 유지합니다. 이미지가 찌그러지는 것을 방지합니다. |

-   **활용 예시**: 캐릭터 초상화, 아이템 아이콘처럼 원본의 비율이 유지되어야 하는 이미지를 표시할 때 `Preserve Aspect` 옵션을 체크하고 사용하면 매우 유용합니다.

#### 2) Sliced

버튼, 패널, 대화창처럼 크기가 유동적으로 변해야 하는 UI를 왜곡 없이 만들기 위한 강력한 모드입니다. 9-slicing 기법을 사용합니다.

**9-slicing 이란?**
스프라이트를 Sprite Editor에서 9개의 조각(네 귀퉁이, 네 변, 중앙)으로 나누는 것입니다. `Sliced` 모드는 이 정보를 바탕으로, 크기가 변할 때 네 귀퉁이는 전혀 스케일링하지 않고, 상하 변은 세로로만, 좌우 변은 가로로만, 그리고 중앙 부분은 가로세로 모두 늘려서 이미지가 깨지지 않게 합니다.

**Public Field**

| 항목 | 설명 |
| :--- | :--- |
| `Fill Center` | `true`이면 중앙 영역을 그리고, `false`이면 비워둡니다. 내용물이 채워질 패널의 배경처럼 사용할 때 `false`로 설정하면 불필요한 렌더링을 줄일 수 있습니다. |
| `Pixels Per Unit Multiplier` | Sliced 또는 Tiled 모드에서 스프라이트 크기를 조절하는 배율입니다. |

-   **사전 준비**: `Sliced` 모드를 사용하려면, 원본 이미지 에셋을 선택하고 인스펙터에서 `Texture Type`을 `Sprite (2D and UI)`로 설정한 뒤, `Sprite Editor`를 열어 `Border` 값을 지정해주어야 합니다.
-   **활용 예시**: 아무리 크기를 늘려도 모서리의 둥근 부분이 깨지지 않는 버튼 배경을 만들 때 필수적입니다. `Simple` 모드로 버튼 크기를 키우면 모서리까지 함께 늘어나 흉하게 찌그러집니다.

#### 3) Tiled

`Sliced`와 비슷하게 9분할된 영역을 사용하지만, 변과 중앙 부분을 늘리는 대신 타일처럼 반복해서 채웁니다.

-   **활용 예시**: 일정한 패턴이 반복되는 배경 이미지를 만들 때 사용합니다. 작은 크기의 반복 가능한 텍스처 하나만 있으면 `Tiled` 모드를 통해 원하는 크기의 영역을 패턴으로 가득 채울 수 있습니다.

#### 4) Filled

스프라이트의 일부분만 점진적으로 채워서 보여주는 방식입니다. 프로그레스 바, 스킬 쿨타임, 원형 게이지 등 동적인 데이터 시각화에 매우 유용합니다.

**Public Field**

| 항목 | 설명 |
| :--- | :--- |
| `Fill Method` | 채워지는 방향을 결정합니다. `Horizontal`(수평), `Vertical`(수직), `Radial 90/180/360`(원형)이 있습니다. |
| `Fill Origin` | 채우기가 시작되는 지점을 결정합니다. (예: `Horizontal`일 경우 `Left` 또는 `Right`) |
| `Fill Amount` | 0에서 1 사이의 값으로, 스프라이트를 얼마나 채워서 보여줄지를 결정합니다. 0은 완전히 비어있고, 1은 완전히 채워진 상태입니다. |
| `Clockwise` | `Radial` 모드에서 채우기가 시계 방향으로 진행될지 반시계 방향으로 진행될지를 결정합니다. |

-   **활용 예시**: 스킬 아이콘 위에 검은색의 반투명한 `Image`를 하나 더 올리고, `Image Type`을 `Filled`, `Fill Method`를 `Radial 360`으로 설정합니다. 스킬 사용 후 스크립트에서 `fillAmount` 값을 1에서 0으로 서서히 줄이면 자연스러운 쿨타임 효과를 만들 수 있습니다.

## 개념 다이어그램 & 코드 예제

```csharp
// HealthBar.cs
// Image의 Filled 타입을 이용해 체력 바를 구현하는 스크립트

using UnityEngine;
using UnityEngine.UI; // Image 컴포넌트를 사용하기 위해 필수

[RequireComponent(typeof(Image))]
public class HealthBar : MonoBehaviour
{
    private Image healthBarImage;

    public float maxHealth = 100f;
    private float currentHealth;

    void Start()
    {
        healthBarImage = GetComponent<Image>();
        
        // 이 스크립트는 Image Type이 Filled일 때만 정상 작동
        if (healthBarImage.type != Image.Type.Filled)
        {
            Debug.LogWarning(this.name + "의 Image Type이 'Filled'가 아닙니다. 정상 작동을 위해 강제로 변경합니다.");
            healthBarImage.type = Image.Type.Filled;
        }
        // 체력 바는 왼쪽에서 오른쪽으로 채워지도록 설정
        healthBarImage.fillMethod = Image.FillMethod.Horizontal;
        healthBarImage.fillOrigin = (int)Image.OriginHorizontal.Left;

        // 시작 시 체력을 가득 채움
        currentHealth = maxHealth;
        UpdateHealthBarUI();
    }

    void Update()
    {
        // 테스트용: 'H' 키를 누르면 체력이 10씩 감소
        if (Input.GetKeyDown(KeyCode.H))
        {
            TakeDamage(10f);
        }
    }

    public void TakeDamage(float damage)
    {
        currentHealth -= damage;
        currentHealth = Mathf.Clamp(currentHealth, 0, maxHealth);
        UpdateHealthBarUI();
    }

    private void UpdateHealthBarUI()
    {
        // 현재 체력 비율을 계산하여 fillAmount에 적용
        healthBarImage.fillAmount = currentHealth / maxHealth;
    }
}
```

## 주요 Public 속성(Property) 및 메서드

| 속성/메서드 | 타입 | 설명 |
| :--- | :--- | :--- |
| `sprite` | `Sprite` | `Image`가 표시할 `Sprite` 에셋입니다. 스크립트에서 아이콘 등을 동적으로 변경할 때 사용합니다. |
| `color` | `Color` | 이미지의 색상 및 투명도입니다. `Color.clear`로 설정하여 이미지를 투명하게 만들 수 있습니다. |
| `fillAmount` | `float` | `Image Type`이 `Filled`일 때, 이미지를 채울 양(0~1)을 나타냅니다. |
| `raycastTarget` | `bool` | 이 이미지가 입력을 받을지 여부를 결정합니다. UI 뒤의 게임 월드를 클릭해야 할 때 `false`로 설정합니다. |
| `preserveAspect` | `bool` | `Simple` 또는 `Filled` 모드에서 이미지의 가로세로 비율을 유지할지 여부를 결정합니다. |
| `overrideSprite` | `Sprite` | (주로 SpriteAtlas와 함께 사용) 원본 스프라이트를 다른 스프라이트로 덮어쓸 때 사용합니다. |
| `SetNativeSize()` | `void` | `RectTransform`의 크기를 `Source Image`의 원본 픽셀 크기로 즉시 재설정합니다. |

## 활용 예시

내용에 따라 크기가 자동으로 변하는 정보창을 `Sliced` 이미지를 이용해 만들어 봅시다.

1.  정보창 배경으로 사용할 이미지 파일을 Unity 프로젝트에 가져옵니다.
2.  해당 이미지 에셋을 선택하고, 인스펙터에서 `Sprite Editor`를 엽니다.
3.  `Border`의 `L, R, T, B` (Left, Right, Top, Bottom) 값을 조절하여 늘어나면 안 되는 모서리 영역을 녹색 선으로 지정하고 `Apply`를 누릅니다.
4.  `Hierarchy`에서 `UI` -> `Image`를 생성하고, 방금 작업한 스프라이트를 `Source Image`에 할당합니다.
5.  `Image Type`을 `Sliced`로 변경합니다.
6.  이제 `RectTransform`의 너비와 높이를 조절해보면, 모서리는 그대로 유지된 채 배경 이미지가 깔끔하게 늘어나는 것을 볼 수 있습니다.
7.  여기에 `Content Size Fitter`와 `Layout Group`을 조합하면, 자식으로 추가되는 텍스트의 길이에 맞춰 정보창의 크기가 자동으로 조절되도록 만들 수 있습니다.

## Note: 주의사항~

-   **Sprite Atlas**: UI에 사용되는 이미지가 많아지면, 이를 `Sprite Atlas`로 묶어 관리하는 것이 성능에 매우 중요합니다. 아틀라스를 사용하면 여러 이미지를 한 번의 드로우 콜(Draw Call)로 처리하여 렌더링 부담을 크게 줄일 수 있습니다.
-   **Sliced Image 사전 작업**: `Sliced` 타입을 사용하려면 반드시 원본 `Sprite` 에셋의 `Sprite Editor`에서 `Border` 값을 설정해야 합니다. 그렇지 않으면 `Simple` 타입과 동일하게 작동하여 이미지가 찌그러집니다.
-   **Raycast Target과 성능**: `Raycast Target`은 상호작용이 필요한 UI(버튼 등)에만 활성화하고, 순수 장식용 배경이나 아이콘 등은 반드시 비활성화하는 것이 좋습니다. 화면의 모든 `Image`가 이 옵션을 켜고 있으면, 매 클릭마다 불필요한 레이캐스트 검사가 늘어나 성능 저하의 원인이 될 수 있습니다.

## Further Study: 확장 학습

-   **Sprite Editor**: 스프라이트를 9-slicing 하거나, 하나의 이미지 파일에서 여러 개의 스프라이트를 추출하는(`Multiple` 모드) 등 `Sprite Editor`의 고급 기능에 대해 학습해보세요.
-   **UI Material & Shader**: `Image` 컴포넌트에 커스텀 머티리얼과 셰이더를 적용하여 그레이스케일 효과, 외곽선 효과, 디졸브 효과 등 독특한 시각적 연출을 구현하는 방법을 연구해보세요.
-   **RawImage Component**: `Image`가 `Sprite`만 표시할 수 있는 것과 달리, `RawImage` 컴포넌트는 모든 종류의 `Texture`를 표시할 수 있습니다. 렌더 텍스처(Render Texture)를 UI에 표시하는 등 더 유연한 사용법이 필요할 때 `RawImage`에 대해 학습해보세요.
