# 사용자와의 상호작용, Button

## Button이란 무엇인가?

현실 세계의 엘리베이터 버튼, 키보드의 키, 스마트폰의 앱 아이콘을 생각해봅시다. 이들의 공통점은 '사용자가 누르면(입력하면) 정해진 동작이 일어난다'는 것입니다. 게임 UI에서 **Button(버튼)** 은 바로 이 역할을 하는 가장 핵심적인 상호작용 요소입니다.

정의하자면, `Button`은 사용자의 클릭(또는 터치) 입력을 감지하여 개발자가 미리 지정해둔 기능(메서드)을 실행하는 UGUI 컴포넌트입니다. '게임 시작', '점프', '아이템 구매', '창 닫기' 등 게임 내에서 일어나는 거의 모든 이벤트의 시작점 역할을 합니다.

`Button`은 보통 눈에 보이는 `Image` 컴포넌트와, 로직을 처리하는 `Button` 컴포넌트, 그리고 버튼의 텍스트를 표시하는 `Text` 또는 `TextMeshPro` 컴포넌트의 조합으로 이루어집니다. 사용자는 이미지를 보고 버튼임을 인지하고, 클릭했을 때 `Button` 컴포넌트가 `On Click()` 이벤트를 발생시켜 실제 동작이 일어나게 됩니다.

## 핵심 요약

-   Button은 사용자의 **클릭(Click) 또는 터치(Touch) 입력**을 받아 특정 기능을 실행하는 상호작용 컴포넌트입니다.
-   **Interactable** 속성으로 버튼의 활성화/비활성화 상태를 제어할 수 있습니다.
-   **Transition** 속성을 통해 사용자의 입력(마우스 올림, 클릭 등)에 따른 시각적 변화를 설정할 수 있습니다. (`Color Tint`, `Sprite Swap`, `Animation`)
-   **On Click ()** 이벤트 리스트에 버튼을 눌렀을 때 실행될 함수(메서드)를 등록하여 사용합니다.

## 세부 개념

### 1. Button 컴포넌트의 주요 속성

**Public Field**

| 항목 | 설명 |
| :--- | :--- |
| `Interactable` | 버튼이 상호작용 가능한 상태인지를 결정합니다. `false`가 되면 버튼을 클릭할 수 없으며, `Transition` 설정에 따라 비활성화된 모습으로 변합니다. (예: 회색으로 변함) |
| `Transition` | 사용자의 입력에 따라 버튼의 모습이 어떻게 변할지를 결정합니다. 아래의 세부 항목에서 자세히 설명합니다. |
| `Navigation` | 키보드나 게임패드로 UI를 조작할 때, 버튼 간의 이동 순서를 어떻게 결정할지를 설정합니다. |

### 2. Transition (전환) 타입

`Transition`은 버튼이 사용자의 마우스 커서나 클릭에 어떻게 반응하는지를 시각적으로 보여주는 중요한 기능입니다.

#### 1) Color Tint (색상 변경)

마우스를 올렸을 때(Highlighted), 클릭했을 때(Pressed), 비활성화되었을 때(Disabled) 등 각 상태에 따라 버튼 `Image`의 색상을 변경합니다. 가장 간단하고 일반적인 방식입니다.

**Public Field**

| 항목 | 설명 |
| :--- | :--- |
| `Target Graphic` | 색상 변경 효과를 적용할 `Image` 컴포넌트를 지정합니다. 보통 버튼 자신을 가리킵니다. |
| `Normal Color` | 평상시 색상입니다. |
| `Highlighted Color` | 마우스 커서를 올렸을 때의 색상입니다. |
| `Pressed Color` | 마우스로 누르고 있을 때의 색상입니다. |
| `Selected Color` | 키보드/패드로 선택되었을 때의 색상입니다. |
| `Disabled Color` | `Interactable`이 `false`일 때의 색상입니다. |
| `Color Multiplier` | 각 상태의 색상을 얼마나 강하게 적용할지 결정하는 배율입니다. |
| `Fade Duration` | 한 상태에서 다른 상태로 색상이 변하는 데 걸리는 시간입니다. |

#### 2) Sprite Swap (스프라이트 교체)

각 상태에 따라 버튼의 `Source Image`를 다른 스프라이트로 교체합니다. 평소 이미지, 마우스를 올렸을 때의 이미지, 클릭했을 때의 이미지를 각각 다르게 디자인하여 훨씬 역동적인 버튼을 만들 수 있습니다.

**Public Field**

| 항목 | 설명 |
| :--- | :--- |
| `Target Graphic` | 스프라이트를 교체할 `Image` 컴포넌트를 지정합니다. |
| `Highlighted Sprite` | 마우스 커서를 올렸을 때 표시될 스프라이트입니다. |
| `Pressed Sprite` | 마우스로 누르고 있을 때 표시될 스프라이트입니다. |
| `Selected Sprite` | 키보드/패드로 선택되었을 때 표시될 스프라이트입니다. |
| `Disabled Sprite` | `Interactable`이 `false`일 때 표시될 스프라이트입니다. |

#### 3) Animation (애니메이션)

가장 자유도가 높은 방식으로, 각 상태에 따라 `Animator` 컴포넌트를 통해 애니메이션을 재생합니다. 버튼이 커졌다 작아지거나, 흔들리거나, 복잡한 파티클 효과가 나타나는 등 원하는 모든 종류의 시각적 연출이 가능합니다. `Animator`와 `Animation`에 대한 이해가 필요합니다.

### 3. On Click () 이벤트

버튼의 존재 이유이자 가장 핵심적인 기능입니다. 이 리스트에 등록된 함수들은 버튼이 클릭될 때마다 순서대로 호출됩니다.

-   **인스펙터에서 등록**:
    1.  `+` 버튼을 눌러 슬롯을 추가합니다.
    2.  실행할 함수가 들어있는 스크립트가 연결된 `GameObject`를 슬롯의 `Object` 필드에 드래그 앤 드롭합니다.
    3.  우측의 `No Function` 드롭다운 메뉴에서 원하는 `public` 함수를 선택합니다.
-   **스크립트에서 등록**: `AddListener` 메서드를 사용하여 코드 상에서 동적으로 함수를 연결하거나 제거(`RemoveListener`)할 수 있습니다.

## 코드 예제

```csharp
// GameManager.cs
// 버튼 클릭 이벤트를 처리하는 다양한 함수를 가진 스크립트

using UnityEngine;
using UnityEngine.UI; // Button을 사용하기 위해 필요
using UnityEngine.SceneManagement; // 씬 전환을 위해 필요

public class GameManager : MonoBehaviour
{
    public Button quitButton;
    private int score = 0;

    void Start()
    {
        // 코드(스크립트)로 OnClick 리스너를 추가하는 방법
        if (quitButton != null)
        {
            // quitButton을 클릭하면 QuitGame 함수가 호출되도록 등록
            quitButton.onClick.AddListener(QuitGame);
            
            // 람다식을 사용하여 간단한 로직을 바로 연결할 수도 있음
            // quitButton.onClick.AddListener(() => Debug.Log("람다식으로 종료!"));
        }
    }

    // 인스펙터의 OnClick()에 연결될 함수 (반드시 public이어야 함)
    public void StartGame()
    {
        SceneManager.LoadScene("GameScene");
    }

    // 인스펙터 또는 코드로 연결될 수 있는 함수
    public void AddScore(int amount)
    {
        score += amount;
        Debug.Log("현재 점수: " + score);
    }

    // 코드로만 연결된 함수
    private void QuitGame()
    {
        Debug.Log("게임을 종료합니다.");
        Application.Quit();
    }
}
```

## 활용 예시

점수를 10점 올리는 버튼을 만들어 봅시다.

1.  `Hierarchy`에서 `UI -> Button - TextMeshPro`를 생성합니다.
2.  버튼의 자식인 `Text` 오브젝트의 내용을 "점수 +10"으로 변경합니다.
3.  `GameManager`와 같은 역할을 할 빈 `GameObject`를 만들고, 위의 `GameManager.cs` 스크립트를 추가합니다.
4.  다시 버튼을 선택하고 인스펙터의 `On Click ()` 리스트에서 `+`를 누릅니다.
5.  `Hierarchy`의 `GameManager` 오브젝트를 `On Click ()` 슬롯에 드래그 앤 드롭합니다.
6.  함수 선택 드롭다운 메뉴에서 `GameManager -> AddScore (int)`를 선택합니다.
7.  `AddScore` 함수 아래에 나타난 입력 필드에 숫자 `10`을 입력합니다.
8.  게임을 실행하고 버튼을 누르면 콘솔 창에 "현재 점수: 10"이 출력됩니다.

## Note: 주의사항

-   **Public 함수만 연결 가능**: 인스펙터의 `On Click ()` 이벤트에는 `public`으로 선언된 함수만 노출되고 연결할 수 있습니다.
-   **매개변수(Parameter) 제약**: 인스펙터에서 연결하는 함수는 매개변수가 없거나, `int`, `float`, `string`, `bool`, `Object` 등 기본 타입의 매개변수 하나만 가질 수 있습니다. 복잡한 데이터를 전달해야 할 때는 람다식이나 별도의 클래스를 통해 `AddListener`로 등록하는 것이 좋습니다.
-   **`AddListener` 중복 방지**: `Start()`나 `Awake()`에서 `AddListener`를 사용할 때는 괜찮지만, `OnEnable()` 등 여러 번 호출될 수 있는 곳에서 사용한다면, `OnDisable()`에서 `RemoveListener`로 해제해주어야 이벤트가 중복 등록되는 것을 막을 수 있습니다.

## Further Study: 확장 학습

-   **Event Trigger**: `Button`이 `On Click` 이벤트만 처리하는 것과 달리, `Event Trigger` 컴포넌트를 사용하면 `Pointer Enter`(마우스 진입), `Pointer Exit`(마우스 이탈), `Drag`(드래그) 등 훨씬 다양한 종류의 입력 이벤트를 감지하고 함수를 연결할 수 있습니다.
-   **Toggle**: '켜고 끄기' 상태를 가지는 버튼입니다. 소리 On/Off, 전체 화면 설정 등 상태 유지가 필요한 옵션 버튼을 만들 때 사용됩니다.
-   **UnityEvent**: `Button`의 `On Click ()`은 `UnityEvent`라는 시스템으로 구현되어 있습니다. 이 `UnityEvent`를 직접 만들고 활용하면, 버튼뿐만 아니라 게임의 모든 곳에서 인스펙터 기반의 유연한 이벤트 시스템을 구축할 수 있습니다.
