# Canvas란?

UI를 만든다는 것을 그림을 그리는 것에 비유해봅시다. 멋진 캐릭터나 배경을 그리려면 가장 먼저 무엇이 필요할까요? 
바로 그림을 그릴 **'캔버스'** 입니다. 아무리 실력이 좋은 화가라도 그림을 그릴 공간이 없다면 아무것도 표현할 수 없죠.

유니티에서 **Canvas** 는 바로 이 화가의 캔버스와 같은 역할을 합니다. 
모든 UI 요소(버튼, 이미지, 텍스트 등)들이 그려지는 기본 바탕이자 무대입니다. 
3D 모델이나 2D 스프라이트가 게임 월드(Game World)에 직접 놓이는 것과 달리, 
모든 UI 요소들은 반드시 Canvas 위에, 그리고 그 자식으로 존재해야만 화면에 나타날 수 있습니다.

정리하자면, **Canvas(캔버스)** 는 모든 UGUI(Unity Graphical User Interface) 요소들을 
담고, 그리고, 관리하는 핵심 컴포넌트입니다. UI를 위한 별도의 렌더링 공간을 제공하며, 
화면 크기 변화에 어떻게 대응할지 결정하는 중요한 기준점이 됩니다.

## 핵심 요약

-   Canvas는 모든 UI 요소(Element)를 담는 **루트(Root) 게임 오브젝트** 입니다.
-   UI 요소는 반드시 Canvas **하위에 존재** 해야 화면에 그려집니다.
-   **Render Mode(렌더 모드)** 설정에 따라 Canvas의 렌더링 방식이 결정됩니다. (Screen Space vs. World Space)
-   **Canvas Scaler(캔버스 스케일러)** 컴포넌트는 화면 크기 변화에 따른 UI 크기 조절 방식을 결정합니다.

## 세부 개념

### 1. Render Mode (렌더 모드)

Canvas가 UI를 화면에 어떻게 그릴지를 결정하는 가장 중요한 속성입니다. 세 가지 모드가 있으며, 각각의 특징과 사용 사례가 명확히 다릅니다.

#### 1) Screen Space - Overlay

가장 일반적으로 사용되는 모드입니다. 이 모드에서 Canvas는 렌더링된 씬(Scene) 위에 마치 투명한 필름을 한 장 덧댄 것처럼 UI를 그립니다. 
즉, 게임 화면의 **가장 위에** UI가 그려지므로 3D 오브젝트나 다른 어떤 요소에도 가려지지 않습니다. 화면 크기가 바뀌면 Canvas 크기도 자동으로 맞춰집니다.

**Public Field**  

| 항목             | 설명                                                                                                      |
|----------------|:--------------------------------------------------------------------------------------------------------|
| `PixelPerfect` | 체크하면 UI 요소들이 픽셀 경계에 정확히 맞춰져 흐릿함 없이 선명하게 렌더링됩니다. 도트 그래픽 스타일의 UI에 유용합니다.                                  |
| `SortingOrder` | 여러 개의 `Screen Space - Overlay` 캔버스가 있을 때, 이 숫자가 높을수록 더 위에 그려집니다. (예: 기본 HUD 캔버스는 0, 그 위에 뜨는 팝업 캔버스는 10) |


#### 2) Screen Space - Camera
이 모드는 지정된 특정 **카메라(Camera)** 로부터 일정한 거리만큼 떨어진 공간에 UI를 그립니다. `Overlay` 모드처럼 화면에 고정되어 보이지만, 카메라 시야각의 영향을 받으며 3D 공간에 배치된 것처럼 원근감을 가집니다. 가장 큰 특징은 카메라와 UI 사이에 다른 3D 오브젝트나 파티클 효과를 배치할 수 있다는 점입니다.

**Public Field**  

| 항목               | 설명                                        |
|------------------|:------------------------------------------|
| `Render Camera`  | UI를 렌더링할 기준 카메라를 지정합니다. **반드시 지정해야 합니다.** |
| `Plane Distance` | Render Camera`로부터 UI 평면이 얼마나 떨어져 있을지를 결정합니다. 이 값이 클수록 UI가 카메라에서 멀어집니다. |
| `Sorting Layer` & `Order in Layer` | 렌더링 순서를 결정합니다. 이를 통해 UI가 특정 스프라이트나 파티클보다 앞에, 혹은 뒤에 그려지도록 제어할 수 있습니다. |

-   **활용 예시**:
    -   **실용 예제 1**: FPS 게임의 조준점(Crosshair)을 이 모드로 구현하고 `Plane Distance`를 1 정도로 가깝게 설정합니다. 그리고 총구 화염(Muzzle Flash) 파티클 효과가 조준점보다 앞에(카메라에 더 가깝게) 렌더링되도록 `Sorting Layer`를 조절하면, 발사 효과가 UI를 잠시 가리는 현실적인 연출이 가능합니다.
    -   **실용 예제 2**: 3D 캐릭터가 화면에 나타나 대사를 하는 연출에서, 캐릭터 모델이 UI 대화창 일부를 가리도록 하여 입체감을 줄 수 있습니다.
-   **잘못된 코드(오류 예시)**: `Render Camera` 필드에 아무것도 할당하지 않는 경우.
    -   **결과**: Canvas가 어느 카메라를 기준으로 그려져야 할지 모르기 때문에 UI가 아예 화면에 나타나지 않습니다.

#### 3) World Space
UI를 3D 공간 안의 일반적인 게임 오브젝트처럼 취급합니다. `Rect Transform`을 사용하지만 일반 `Transform`처럼 위치, 회전, 크기 조절이 가능하며, 다른 3D 오브젝트에 가려지거나 함께 움직일 수 있습니다.

**Public Field**

| 항목               | 설명                                                                                                                            |
|------------------|:------------------------------------------------------------------------------------------------------------------------------|
| `Event Camera`| 이 캔버스 위의 UI 요소들(버튼 등)과 상호작용(클릭, 터치 등)을 감지할 카메라를 지정합니다. 보통 플레이어의 시점을 나타내는 `Main Camera`를 할당합니다. **이걸 설정하지 않으면 버튼이 눌리지 않습니다.**  |

-   **활용 예시**:
    -   **실용 예제 1**: MMORPG에서 다른 플레이어나 NPC의 머리 위에 이름과 체력 바를 띄울 때. 캐릭터가 움직이면 UI도 함께 따라 움직입니다.
    -   **실용 예제 2**: VR/AR 콘텐츠에서 사용자가 직접 조작해야 하는 3D 컨트롤 패널이나 정보창을 구현할 때.
-   **잘못된 코드(오류 예시)**: `Event Camera`를 할당하지 않고 `World Space` 캔버스에 버튼을 배치한 경우.
    -   **결과**: 버튼이 화면에는 보이지만, 유니티의 이벤트 시스템이 어떤 카메라의 시점에서 레이캐스트(Raycast)를 쏴야 할지 몰라 사용자의 클릭 입력을 감지하지 못합니다.

### 2. Canvas Scaler (캔버스 스케일러)

Canvas에 자식으로 함께 생성되는 컴포넌트로, 다양한 해상도의 화면에서 UI가 어떻게 보일지를 결정하는 중요한 역할을 합니다.

#### 1) UI Scale Mode

**Public Field**

| 항목                       | 설명                                                                                                                                    |
|--------------------------|:--------------------------------------------------------------------------------------------------------------------------------------|
| `Constant Pixel Size`    | UI 요소의 픽셀 크기를 고정합니다. 100x50 픽셀 크기의 버튼은 4K 해상도 모니터에서는 작게 보이고, 저해상도 모바일 기기에서는 화면을 크게 차지하게 됩니다. 거의 사용되지 않습니다.                            |
| `Scale With Screen Size` | **가장 중요하고 널리 쓰이는 모드**입니다. 기준 해상도(Reference Resolution)를 설정하고, 실제 게임 화면 해상도가 이와 달라질 때 UI 요소들의 크기를 자동으로 조절합니다.                          |
| `Reference Resolution`   | UI를 디자인할 기준이 되는 해상도입니다. (예: 1920 x 1080)                                                                                              |
| `Constant Pixel Size`    | 이 캔버스 위의 UI 요소들(버튼 등)과 상호작용(클릭, 터치 등)을 감지할 카메라를 지정합니다. 보통 플레이어의 시점을 나타내는 `Main Camera`를 할당합니다. **이걸 설정하지 않으면 버튼이 눌리지 않습니다.**          |
| `Screen Match Mode`      | 화면 비율이 기준과 달라질 때 너비와 높이 중 어느 쪽을 기준으로 스케일을 맞출지 결정합니다.                                                                                  |
| `Match Width Or Height`  | 슬라이더로 너비(0)와 높이(1) 사이의 가중치를 조절합니다. 0.5로 설정하면 너비와 높이의 중간 비율로 조절되어 대부분의 상황에서 유연하게 대처할 수 있습니다.                                           |
| `Constant Physical Size` | UI 요소가 화면 해상도와 상관없이 실제 물리적 크기(인치, 센티미터 등)를 유지하도록 시도합니다. 화면의 DPI(Dots Per Inch) 정보를 사용하며, 사용자의 손가락 크기에 맞춰야 하는 버튼 등 특수한 경우에 사용될 수 있습니다. |

```csharp
// CanvasActivator.cs
// 두 개의 다른 Canvas를 관리하여 게임 상태에 따라 전환하는 스크립트

using UnityEngine;

public class CanvasActivator : MonoBehaviour
{
    // 인스펙터에서 게임 HUD 캔버스와 설정 메뉴 캔버스를 할당
    public Canvas gameHudCanvas;
    public Canvas settingsMenuCanvas;

    void Start()
    {
        // 게임 시작 시에는 HUD만 보이도록 설정
        gameHudCanvas.gameObject.SetActive(true);
        settingsMenuCanvas.gameObject.SetActive(false);
    }

    void Update()
    {
        // 'ESC' 키를 누르면 설정 메뉴를 토글
        if (Input.GetKeyDown(KeyCode.Escape))
        {
            // 현재 설정 메뉴의 활성화 상태를 반전시킴
            bool isSettingsActive = !settingsMenuCanvas.gameObject.activeSelf;
            settingsMenuCanvas.gameObject.SetActive(isSettingsActive);

            // 설정 메뉴가 활성화되면 HUD는 비활성화, 반대도 마찬가지
            gameHudCanvas.gameObject.SetActive(!isSettingsActive);
        }
    }
}
```

## 주요 Public 속성(Property)

| 속성 (Property) | 타입 | 설명 | 대표 활용 시나리오 |
| :--- | :--- | :--- | :--- |
| `renderMode` | `RenderMode` | 캔버스의 렌더링 모드를 가져오거나 설정합니다. | 게임 상태에 따라 UI를 월드 스페이스에서 스크린 스페이스로 전환할 때 사용합니다. |
| `isRootCanvas` | `bool` | 이 캔버스가 캔버스 계층의 루트인지 여부를 반환합니다. (읽기 전용) | 중첩된 캔버스 구조에서 최상위 캔버스를 찾을 때 사용합니다. |
| `worldCamera` | `Camera` | 캔버스에 의해 사용되는 카메라를 가져오거나 설정합니다. (`Screen Space - Camera` 또는 `World Space` 모드에서 사용) | `World Space` 캔버스의 UI 이벤트를 처리할 카메라를 지정하거나, `Screen Space - Camera`의 렌더링 기준 카메라를 변경할 때 사용합니다. |
| `sortingOrder` | `int` | 동일한 `Sorting Layer` 내에서 이 캔버스의 렌더링 순서를 결정합니다. 숫자가 높을수록 위에 그려집니다. | 여러 개의 `Screen Space - Overlay` 캔버스가 있을 때, 팝업 캔버스를 항상 다른 UI 위에 표시하고 싶을 때 사용합니다. |
| `pixelPerfect` | `bool` | 픽셀 퍼펙트 렌더링을 활성화/비활성화합니다. | 도트 그래픽 기반 게임에서 UI가 흐릿해지지 않고 선명하게 보이도록 할 때 사용합니다. |
| `overrideSorting`| `bool` | 중첩된 캔버스일 경우, 부모 캔버스의 정렬 순서를 무시하고 자신만의 `Sorting Layer`와 `Order in Layer`를 사용하도록 강제합니다. | 복잡한 UI에서 특정 UI 그룹(예: 미니맵)이 다른 UI 그룹과 독립적인 렌더링 순서를 갖게 하고 싶을 때 사용합니다. |

## 활용 예시

게임에 '일시정지 메뉴'를 만드는 간단한 과정을 통해 Canvas의 기본 활용법을 익혀봅시다.

1.  **Canvas 생성**: `Hierarchy` 창에서 마우스 오른쪽 클릭 -> `UI` -> `Canvas`를 선택하여 새로운 Canvas를 생성합니다.
2.  **Render Mode 설정**: 생성된 Canvas의 인스펙터(Inspector) 창에서 `Render Mode`가 `Screen Space - Overlay`로 설정된 것을 확인합니다.
3.  **Canvas Scaler 설정**: `Canvas Scaler` 컴포넌트의 `UI Scale Mode`를 `Scale With Screen Size`로 변경하고, `Reference Resolution`을 `X: 1920`, `Y: 1080`으로 설정합니다. `Screen Match Mode`는 `Match Width Or Height`로 두고 슬라이더를 `0.5`에 맞춥니다.
4.  **패널 추가**: Canvas 하위에 메뉴의 배경이 될 패널을 추가합니다. `Hierarchy`에서 Canvas를 우클릭하고 `UI` -> `Panel`을 선택합니다.
5.  **버튼 추가**: '게임으로 돌아가기' 기능을 할 버튼을 Panel 하위에 추가합니다. `Panel` 우클릭 -> `UI` -> `Button - TextMeshPro`를 선택합니다.
6.  **스크립트 작성**: 위 `코드 예제`의 `CanvasActivator`와 같은 스크립트를 작성하여, 특정 키(예: `ESC`)를 눌렀을 때 일시정지 메뉴 Canvas(또는 Panel)가 활성화/비활성화되도록 로직을 구현합니다.

이제 여러분은 Canvas의 기본을 이해했습니다. 다음 실습 문제를 통해 직접 다양한 UI를 구성해보면서 개념을 확실히 다져봅시다.

## Note: 주의사항

-   **UI 재빌드(Rebuild) 최적화**: 하나의 Canvas에 너무 많은 UI 요소를 배치하면, 그중 단 하나의 요소만 변경되어도 Canvas 전체가 다시 그려져야 하는 '재빌드' 과정에서 성능 저하가 발생할 수 있습니다. 자주 변하는 UI(예: 체력 바, 타이머)와 거의 변하지 않는 UI(예: 메뉴 배경)는 별도의 Canvas로 분리하여 관리하는 것이 성능 최적화에 유리합니다.
-   **Event Camera 할당**: `World Space` Canvas에서 버튼 클릭과 같은 상호작용을 구현하려면, 반드시 인스펙터의 `Event Camera` 속성에 이벤트를 감지할 카메라(보통 `Main Camera`)를 할당해야 합니다. 이를 잊으면 UI가 반응하지 않습니다.
-   **Graphic Raycaster**: Canvas를 생성하면 `Graphic Raycaster` 컴포넌트가 자동으로 부착됩니다. 이 컴포넌트는 마우스나 터치 입력을 감지하여 어떤 UI 요소가 클릭되었는지 판별하는 중요한 역할을 합니다. 실수로 삭제하지 않도록 주의해야 합니다.

## Further Study: 확장 학습

-   **Nested Canvases (중첩 캔버스)**: 캔버스 안에 또 다른 캔버스를 넣어 UI 렌더링 순서와 재빌드 단위를 더 세밀하게 제어하는 기법에 대해 학습해보세요.
-   **Custom Graphic Raycasters**: 기본 `Graphic Raycaster` 외에, 3D 오브젝트나 2D 물리 시스템과 상호작용하는 커스텀 Raycaster를 구현하는 방법을 연구해보세요.
-   **Unity UI Profiling**: 유니티 프로파일러(Profiler)를 사용하여 UI 성능을 분석하고 병목 현상을 찾아내는 방법을 학습하여 복잡한 UI의 성능을 최적화해보세요.
