# 사용자로부터 텍스트를 입력받는, Input Field

## Input Field란?

웹사이트의 로그인 창, 검색 엔진의 검색어 입력창, 스마트폰의 메시지 앱을 떠올려봅시다. 이들은 모두 사용자가 키보드를 통해 직접 글자나 숫자를 입력할 수 있는 공간을 제공합니다. UI에서 **Input Field(입력 필드)** 는 바로 이러한 '사용자 텍스트 입력' 기능을 담당하는 컴포넌트입니다.

정의하자면, `Input Field`는 사용자가 런타임(게임 실행 중)에 키보드를 통해 문자열(string)을 입력하고 수정할 수 있도록 해주는 상호작용 컴포넌트입니다. 플레이어의 이름 입력, 채팅 메시지 전송, 비밀번호 입력 등 사용자로부터 텍스트 정보를 받아야 하는 모든 경우에 사용됩니다.

`Input Field`는 보통 사용자가 입력하는 텍스트를 보여주는 `Text` 영역과, 필드의 의미를 알려주는 안내 문구인 `Placeholder`로 구성됩니다. (Unity 2021 이후 버전부터는 `TextMeshPro - Input Field`가 기본입니다.)

## 핵심 요약

-   Input Field는 사용자로부터 **키보드 입력을 받아 텍스트(문자열)를 처리**하는 컴포넌트입니다.
-   **Text** 속성을 통해 현재 입력된 문자열을 스크립트에서 가져오거나 설정할 수 있습니다.
-   **Content Type** 설정을 통해 입력 가능한 문자의 종류를 제한할 수 있습니다. (예: 숫자, 이메일 주소, 비밀번호)
-   **On Value Changed()** 와 **On End Edit()** (또는 `onSubmit`) 이벤트를 통해 입력 내용이 변경되거나 입력이 완료되었을 때 특정 함수를 호출할 수 있습니다.

## 세부 개념

### 1. Input Field (TextMeshPro) 컴포넌트의 주요 속성

`TMP_InputField` 컴포넌트는 다양한 속성을 통해 입력 방식과 내용을 세밀하게 제어할 수 있습니다.

**Public Field (TMP_InputField)**

| 항목 | 설명 |
| :--- | :--- |
| `Text Viewport` | 실제 텍스트가 표시되는 영역의 `RectTransform`을 지정합니다. |
| `Text Component` | 텍스트를 렌더링할 `TextMeshProUGUI` 컴포넌트를 지정합니다. |
| `Placeholder` | 입력된 텍스트가 없을 때 대신 표시될 안내 문구(예: "이름을 입력하세요...")를 담당하는 `Graphic` 컴포넌트(보통 `TextMeshProUGUI`)를 지정합니다. |
| `Character Limit` | 입력 가능한 최대 글자 수를 제한합니다. 0이면 무제한입니다. |
| `Content Type` | 입력 가능한 콘텐츠의 종류를 미리 정의된 형식으로 제한합니다. |
| `Line Type` | 줄바꿈 처리 방식을 결정합니다. `Single Line`(한 줄), `Multi Line Submit`(Enter로 제출), `Multi Line Newline`(Enter로 줄바꿈) |
| `text` | `string` 타입의 속성으로, 현재 입력 필드에 들어있는 실제 텍스트 값입니다. 스크립트에서 이 값을 읽어 사용하거나, 이 값을 변경하여 필드의 내용을 바꿀 수 있습니다. |

### 2. Content Type (콘텐츠 타입)

입력 데이터의 유효성을 검사하는 번거로움을 크게 줄여주는 매우 유용한 기능입니다.

| 항목 | 설명 |
| :--- |:---|
|`Standard`|  모든 문자를 입력할 수 있습니다.  |
|`Integer Number`|정수만 입력할 수 있습니다.|
|`Decimal Number`|정수와 소수를 입력할 수 있습니다.|
|`Alphanumeric`|영문자와 숫자만 입력할 수 있습니다.|
|`Name`|이름에 주로 쓰이는 형식(첫 글자 대문자 등)을 자동으로 적용합니다.|
|`Email Address`|이메일 주소 형식에 맞는지 기본적인 유효성을 검사합니다.|
|`Password`|입력된 모든 문자를 `*` (별표)로 표시하여 가려줍니다. `Asterisk Character` 속성에서 표시 문자를 변경할 수 있습니다.|

### 3. 주요 이벤트

-   **On Value Changed (String)**: 입력 필드의 **내용이 한 글자라도 바뀔 때마다** 실시간으로 호출됩니다. 실시간 검색어 제안, 글자 수 표시 등에 유용합니다.
-   **On End Edit (String)**: 사용자가 입력 필드 외부를 클릭하거나, `Enter` 키를 눌러 **입력을 마쳤을 때** 한 번 호출됩니다. 로그인, 메시지 전송 등 최종적인 입력값을 처리할 때 주로 사용됩니다. (Unity 2021 이후 버전에서는 `onSubmit`과 `onDeselect` 등으로 세분화되었습니다.)

## 코드 예제

```csharp
// LoginManager.cs
// Input Field를 사용해 아이디와 비밀번호를 입력받는 간단한 로그인 시스템

using UnityEngine;
using TMPro; // TextMeshPro 컴포넌트를 사용하기 위해 필수

public class LoginManager : MonoBehaviour
{
    public TMP_InputField idInputField;
    public TMP_InputField passwordInputField;
    public TextMeshProUGUI feedbackText;

    private string correctId = "Admin";
    private string correctPassword = "1234";

    void Start()
    {
        // 비밀번호 필드의 Content Type을 Password로 설정 (스크립트에서도 가능)
        passwordInputField.contentType = TMP_InputField.ContentType.Password;
        passwordInputField.ForceLabelUpdate(); // 변경사항 즉시 적용

        // 입력 완료 이벤트에 리스너 등록
        idInputField.onEndEdit.AddListener(OnInputEnd);
        passwordInputField.onEndEdit.AddListener(OnInputEnd);
    }

    // 입력이 끝났을 때 호출될 함수
    private void OnInputEnd(string text)
    {
        Debug.Log("입력 완료: " + text);
    }

    // '로그인' 버튼의 OnClick 이벤트에 연결될 함수
    public void OnLoginButtonClick()
    {
        // Input Field의 현재 text 값을 가져와서 확인
        string inputId = idInputField.text;
        string inputPassword = passwordInputField.text;

        if (inputId == correctId && inputPassword == correctPassword)
        {
            feedbackText.color = Color.green;
            feedbackText.text = "로그인 성공!";
        }
        else
        {
            feedbackText.color = Color.red;
            feedbackText.text = "아이디 또는 비밀번호가 일치하지 않습니다.";
            
            // 실패 시 비밀번호 필드 초기화
            passwordInputField.text = "";
        }
    }
}
```

## 활용 예시

플레이어의 이름을 입력받아 저장하는 기능을 만들어 봅시다.

1.  `Hierarchy`에서 `UI -> Input Field - TextMeshPro`를 생성합니다.
2.  `Placeholder`의 텍스트를 "이름을 입력하세요 (최대 10자)"로 변경합니다.
3.  `Input Field` 컴포넌트의 `Character Limit`을 `10`으로 설정합니다.
4.  `UI -> Button - TextMeshPro`를 생성하고 "확인"이라고 텍스트를 변경합니다.
5.  아래와 같은 함수를 가진 스크립트를 만듭니다.
    ```csharp
    public class PlayerNameSetter : MonoBehaviour
    {
        public TMP_InputField nameInputField;

        public void SavePlayerName()
        {
            string playerName = nameInputField.text;
            if (string.IsNullOrWhiteSpace(playerName))
            {
                Debug.LogWarning("이름이 비어있습니다!");
                return;
            }
            
            // PlayerPrefs 등을 사용해 플레이어 이름을 기기에 저장
            PlayerPrefs.SetString("PlayerName", playerName);
            PlayerPrefs.Save();
            
            Debug.Log("플레이어 이름 '" + playerName + "' 이(가) 저장되었습니다.");
        }
    }
    ```
6.  "확인" 버튼의 `On Click()` 이벤트에 위 스크립트의 `SavePlayerName()` 함수를 연결합니다.
7.  게임을 실행하고 이름을 입력한 뒤 버튼을 누르면, 입력한 이름이 콘솔에 출력되고 기기에 저장됩니다.

## Note: 주의사항

-   **모바일 환경에서의 키보드**: 모바일 기기에서는 `Input Field`를 터치하면 자동으로 시스템 키보드가 나타납니다. `Content Type`에 따라 나타나는 키보드의 종류(일반, 숫자, 이메일 등)가 달라지므로 적절히 설정하는 것이 사용자 경험에 매우 중요합니다.
-   **보안**: 비밀번호와 같은 민감한 정보를 다룰 때는 `Content Type`을 `Password`로 설정하는 것이 기본이며, 실제 서비스에서는 서버로 전송 시 반드시 암호화 처리를 해야 합니다.
-   **`text` 속성과 이벤트**: 스크립트로 `inputField.text = "..."` 코드를 실행하여 내용을 변경하는 경우에는 `On Value Changed` 이벤트가 발생하지 않습니다. 이 이벤트는 사용자의 키보드 입력에 의해서만 트리거됩니다.

## Further Study: 확장 학습

-   **TMP_InputValidator**: `TextMeshPro`의 `Input Field`는 `TMP_InputValidator`라는 커스텀 스크립트를 상속받아 자신만의 입력 유효성 검사 규칙을 만들 수 있습니다. 예를 들어, '한글만 입력 가능' 또는 '특수문자 제외'와 같은 복잡한 규칙을 구현할 수 있습니다.
-   **Caret (캐럿)**: 입력 필드에서 깜빡이는 커서를 '캐럿'이라고 합니다. `TMP_InputField`의 `Caret Blink Rate`, `Caret Color` 등의 속성을 통해 캐럿의 모양과 동작을 커스터마이징할 수 있습니다.
-   **모바일 키보드 제어**: `TouchScreenKeyboard` 클래스를 사용하면 스크립트에서 직접 모바일 키보드를 제어하고, 입력된 값을 받아오는 등 더 낮은 수준의 제어가 가능합니다.
